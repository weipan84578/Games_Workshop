# 🐍 Web 版貪食蛇遊戲規格書

**文件版本：** v1.0  
**建立日期：** 2026-04-23  
**適用平台：** 桌面瀏覽器 / 手機瀏覽器  

---

## 1. 專案概述

### 1.1 專案目標

開發一款可在現代瀏覽器上運行的貪食蛇（Snake）網頁遊戲，支援鍵盤與觸控操作，提供流暢的遊戲體驗與排行榜功能。

### 1.2 目標使用者

- 休閒玩家（所有年齡層）
- 行動裝置使用者
- 想挑戰高分的競技玩家

### 1.3 技術限制

- 純前端實作（HTML5 + CSS3 + JavaScript）
- 不依賴任何第三方遊戲框架
- 支援主流瀏覽器：Chrome、Firefox、Safari、Edge（最新兩個版本）
- 無需後端伺服器（排行榜可選用 localStorage 或後端 API）

## 2. 功能需求

### 2.1 核心遊戲功能

#### 2.1.1 遊戲地圖

| 屬性 | 規格 |
|------|------|
| 渲染方式 | HTML5 Canvas |
| 預設格子大小 | 20×20 像素 |
| 預設地圖尺寸 | 20×20 格（400×400px） |
| 邊界類型 | 可設定（穿越式 / 撞牆死亡） |
| 背景色 | 深色底，格線可選顯示 |

#### 2.1.2 蛇（Snake）

- **初始長度：** 3 格
- **初始位置：** 地圖中央
- **初始方向：** 向右
- **移動速度：**
  - 初始：每 150ms 移動一格
  - 每吃 5 顆食物加快 10ms（最低 60ms）
- **死亡條件：**
  - 撞到自身身體
  - 撞到邊界（穿越模式除外）
- **視覺呈現：**
  - 頭部與身體顏色區分
  - 頭部顯示眼睛方向動畫
  - 身體段落有圓角與漸層效果

#### 2.1.3 食物（Food）

| 類型 | 出現機率 | 分數 | 特效 |
|------|----------|------|------|
| 一般食物（紅色） | 70% | +10 | 蛇增長 1 格 |
| 加速食物（黃色） | 15% | +20 | 蛇增長 1 格，暫時加速 3 秒 |
| 黃金食物（金色） | 10% | +50 | 蛇增長 1 格，10 秒後消失 |
| 縮短食物（藍色） | 5% | +30 | 蛇縮短 2 格（最短保留 3 格） |

- 食物隨機出現在地圖空白格子上
- 黃金食物有倒數計時閃爍動畫
- 同一時間地圖上最多存在 2 顆食物

#### 2.1.4 分數系統

- 基礎得分依食物類型
- **連擊加成：** 3 秒內連續吃到食物，分數乘以 1.5x（最高 3x）
- **時間加成：** 關卡存活時間越長，每分鐘額外 +100 分
- **最高分記錄** 存於 localStorage

### 2.2 遊戲模式

#### 2.2.1 經典模式（Classic）

- 無時間限制，吃到撞死為止
- 支援難度選擇：簡單 / 普通 / 困難
- 難度影響初始速度與邊界設定

#### 2.2.2 限時模式（Time Attack）

- 60 秒內盡量累積最高分
- 時間條顯示於畫面頂部
- 每吃 5 顆食物延長 5 秒（上限 90 秒）

#### 2.2.3 障礙物模式（Obstacle）

- 地圖上隨機分布固定障礙物（牆壁）
- 障礙物數量隨關卡遞增
- 共 10 關，通關後顯示完成畫面

### 2.3 控制方式

#### 2.3.1 鍵盤控制

| 按鍵 | 功能 |
|------|------|
| ↑ / W | 向上 |
| ↓ / S | 向下 |
| ← / A | 向左 |
| → / D | 向右 |
| Space | 暫停 / 繼續 |
| R | 重新開始 |
| Esc | 返回主選單 |

#### 2.3.2 觸控控制（手機）

- 滑動手勢（Swipe）控制方向
- 虛擬搖桿（可在設定中切換）
- 螢幕左/右半部點擊可選擇性左右轉

#### 2.3.3 防誤觸設計

- 禁止 180 度反向操作（如向右時不可直接向左）
- 連續快速輸入排隊處理（Queue Buffer，最多 2 個）

### 2.4 UI 畫面規格

#### 2.4.1 主選單（Main Menu）

- 遊戲標題動畫（蛇形文字效果）
- 按鈕：開始遊戲 / 選擇模式 / 設定 / 排行榜
- 顯示個人最高分

#### 2.4.2 遊戲畫面（Game Screen）

```
┌─────────────────────────────────┐
│  分數: 1250   最高: 3200  ⏱ 45s │
│  模式: 限時   難度: 普通         │
├─────────────────────────────────┤
│                                 │
│         [ Canvas 遊戲區 ]       │
│                                 │
├─────────────────────────────────┤
│  [暫停]              [重新開始]  │
└─────────────────────────────────┘
```

#### 2.4.3 暫停畫面（Pause Screen）

- 半透明遮罩覆蓋遊戲畫面
- 按鈕：繼續 / 重新開始 / 返回選單

#### 2.4.4 遊戲結束畫面（Game Over Screen）

- 顯示本局分數、最高分、等級
- 是否破紀錄動畫特效
- 按鈕：再玩一次 / 查看排行榜 / 返回選單

#### 2.4.5 設定畫面（Settings）

| 設定項目 | 選項 |
|----------|------|
| 音效 | 開 / 關 |
| 音樂 | 開 / 關 |
| 格線顯示 | 開 / 關 |
| 邊界模式 | 穿越 / 撞牆死亡 |
| 控制方式（手機） | 滑動 / 虛擬搖桿 |
| 主題 | 深色 / 淺色 / 霓虹 |

#### 2.4.6 排行榜（Leaderboard）

- 顯示前 10 名
- 欄位：排名、暱稱、分數、模式、日期
- 支援分頁（依模式分類）
- 資料儲存於 localStorage（本地）

### 5.2 核心類別設計

#### Snake 類別

```javascript
class Snake {
  constructor(gridX, gridY, direction)
  move()                    // 移動蛇頭，更新身體
  grow(count = 1)           // 增加蛇身長度
  shrink(count = 2)         // 縮短蛇身
  checkSelfCollision()      // 檢查自撞
  checkWallCollision(grid)  // 檢查撞牆
  getHead()                 // 回傳蛇頭座標
  getBody()                 // 回傳所有身體座標
  setDirection(dir)         // 設定移動方向（含防反向邏輯）
}
```

#### Game 類別

```javascript
class Game {
  constructor(config)
  start()                   // 開始遊戲
  pause() / resume()        // 暫停/繼續
  reset()                   // 重置遊戲狀態
  update()                  // 每 tick 更新邏輯
  render()                  // 呼叫 Renderer
  spawnFood()               // 生成食物
  checkCollisions()         // 碰撞偵測
  updateScore(points)       // 更新分數與連擊
  gameOver()                // 結束流程
}
```

#### Renderer 類別

```javascript
class Renderer {
  constructor(canvas, theme)
  drawGrid()                // 繪製格線
  drawSnake(snake)          // 繪製蛇
  drawFood(foods)           // 繪製食物
  drawObstacles(obstacles)  // 繪製障礙物
  drawEffect(type, pos)     // 繪製特效（粒子等）
  clear()                   // 清空畫布
}
```

### 5.3 遊戲迴圈

```
GameLoop:
  requestAnimationFrame(loop)
  ├── elapsed = now - lastTime
  ├── if (elapsed >= tickInterval):
  │     update()           // 邏輯更新
  │     lastTime = now
  └── render()             // 每幀渲染（插補動畫）
```

### 5.4 響應式設計

- Canvas 依據視窗大小動態縮放（保持長寬比）
- 手機橫向模式：地圖縮小，UI 佈局調整
- 最小支援解析度：320×480px
- 斷點設定：
  - Mobile：< 768px
  - Desktop：≥ 768px

### 5.5 效能目標

| 指標 | 目標值 |

## 9. 測試需求

### 9.1 功能測試

- 蛇移動方向正確性（含防反向）
- 各類型食物生成比例
- 碰撞偵測準確性（自撞、撞牆、穿越邊界）
- 分數計算正確性（含連擊倍率）
- localStorage 讀寫正確性

### 9.2 效能測試

- 60fps 穩定性（使用 Chrome DevTools）
- 長時間遊戲（10 分鐘以上）記憶體洩漏測試

### 9.3 跨裝置測試

- 各主流瀏覽器視覺一致性
- 手機觸控靈敏度測試
- 橫/直屏切換行為
