# 四連棋 (Connect Four) 遊戲規格書

版本：v1.0
文件性質：開發規格書（尚未實作，僅供後續開發參考）
目標平台：純前端網頁（Web / 行動裝置瀏覽器）

---

## 1. 專案概述

本專案為一款可離線執行的四連棋網頁遊戲，使用者只需雙擊 `index.html` 即可開啟遊戲，**不需要任何建置工具（build tool）、套件安裝（npm install）或本地伺服器（server）**。遊戲需支援電腦瀏覽器與手機瀏覽器兩種操作情境，並提供多國語系、多種配色主題、可調整難度的 AI 對戰、輕快鋼琴 BGM 與清脆音效。

### 1.1 核心設計原則
- **Zero-Build**：純 HTML + CSS + JavaScript（Vanilla JS），不使用任何需要編譯的框架（React/Vue 需編譯打包，故不採用；如需元件化可用原生 Web Components）。
- **離線優先**：所有資源（字體、音樂、圖示）皆存放於本地資料夾，不依賴外部 CDN，確保雙擊即玩。
- **RWD 優先**：同一套程式碼要能在桌機、平板、手機上都順暢遊玩。
- **高可讀性**：大字體、高對比配色、清楚圖示。

## 2. 技術架構

| 項目 | 技術 |
|---|---|
| 標記語言 | HTML5 |
| 樣式 | CSS3（Flexbox / Grid、CSS 變數做主題色） |
| 邏輯 | JavaScript ES6+（原生，不使用 TypeScript 以避免需要編譯） |
| 模組化 | ES Module（`<script type="module">`），依賴 `index.html` 直接以 `<script>` 與 `<link>` 引入各分類檔案 |
| 儲存 | `localStorage`（記錄進度、設定、語系、音量、勝率統計） |
| 音訊 | HTML5 `<audio>` + Web Audio API（`GainNode` 控制音量，避免直接暴力提升音量造成破音） |
| 圖示 | SVG（向量圖示，任何解析度不失真，可愛插畫風） |
| 字型 | 本地內嵌 Web Font（.woff2，需圓潤可愛且清晰易讀，如圓體 / 手寫感圓潤字體），中日英三語皆需涵蓋 |

> **不需要**：Webpack、Vite、Node.js、npm、Babel、任何 Server-Side 程式。所有檔案以相對路徑引入，直接用瀏覽器開啟 `file://.../index.html` 即可運作。

## 4. 主畫面規格（Main Menu）

### 4.1 需求對應（第 5、14 點）
主畫面**僅呈現四個核心功能按鈕**，維持簡潔乾淨，不放任何設定選項：

| 按鈕 | 說明 |
|---|---|
| 🎮 開始遊戲 | 進入「選擇對戰模式」（人 vs AI／人 vs 人）與「選擇難度」子畫面後開局 |
| ▶️ 繼續遊戲 | 讀取 `localStorage` 中未完成的對局；若無存檔則此按鈕顯示為灰階不可點擊狀態，並提示「目前無進行中的對局」 |
| 📖 遊戲說明 | 開啟說明頁面（Modal 或獨立畫面） |
| ⚙️ 設定 | 開啟設定頁面（語言、主題、音量、難度預設值等） |

### 4.2 視覺配置
- 背景使用溫馨插畫（棋盤桌面情境，暖色調），並疊加半透明遮罩確保按鈕文字對比足夠。
- 四個按鈕垂直置中排列（手機）／置中卡片排列（桌機），按鈕之間留有足夠間距避免誤觸。
- 標題採用大字體 + 可愛圖示（棋子造型的 Logo）。
- 進入頁面時有淡入＋輕微彈跳動畫，營造活潑感。

## 5. RWD 響應式設計規格（第 2、8 點）

### 5.1 斷點設計

| 裝置類型 | 寬度範圍 | 版面策略 |
|---|---|---|
| 手機直式 | < 480px | 單欄，棋盤置頂並自動縮放至可視安全區域，操作按鈕移至畫面底部安全區（不重疊棋盤），採用大顆觸控熱區（最小 44x44px） |
| 手機橫式 / 平板直式 | 480px–1024px | 棋盤置中放大，操作列移至側邊或底部細列 |
| 桌機 / 平板橫式 | > 1024px | 棋盤置中，左右兩側顯示玩家資訊卡（頭像、比分、目前回合提示），操作按鈕固定於棋盤下方，不覆蓋棋盤 |

### 5.2 關鍵規則（避免遮擋遊戲畫面）
- 使用 CSS `env(safe-area-inset-*)` 處理 iPhone 瀏海與底部手勢列安全區域。
- 遊戲內操作按鈕（悔棋、重新開始、回主選單、暫停）一律採用**懸浮於畫面邊緣、半透明底色的小型工具列**，不得覆蓋在棋盤格子正上方；行動裝置直式模式下工具列固定於**畫面最底部**並保留與棋盤至少 12px 的視覺間距。
- 棋盤本身使用 `aspect-ratio: 7 / 6` 搭配 `clamp()` 動態計算格子大小，確保任何螢幕比例下棋盤都完整可見、不需捲動（scroll）即可看到整個棋盤。
- 彈窗（說明／設定／結算）在手機上以全螢幕方式呈現，桌機上以置中卡片呈現，避免小螢幕上彈窗被裁切。
- 橫向與直向切換時（`orientationchange` 事件）即時重新計算棋盤與按鈕位置，過渡以 0.2 秒動畫平滑呈現，不可白屏或跳動。

## 6. 字體與配色規範（第 3、11 點）

### 6.1 字體規範
- 全站字體大小基準值提高：內文最小 18px（桌機）／16px（手機），標題級距 24px～40px。
- 使用 CSS 變數統一管理，範例：
  ```css
  :root {
    --font-size-body: clamp(16px, 2vw, 20px);
    --font-size-title: clamp(24px, 5vw, 40px);
    --font-size-button: clamp(18px, 3vw, 24px);
    --font-weight-emphasis: 700;
  }
  ```
- 字體需清晰明確、筆畫圓潤（避免過細字重造成手機螢幕不易閱讀），中日英文皆選用視覺重量相近的字型搭配，確保切換語系不會有字體不一致的違和感。

### 6.2 配色主題（可於設定頁切換，多樣選擇）
| 主題名稱 | 主色調 | 特色 |
|---|---|---|
| 經典木質 Classic | 木棕＋米白＋深藍棋子／鮮黃棋子 | 仿實體棋盤質感 |
| 海洋藍 Ocean | 天藍＋白＋珊瑚橘棋子／薄荷綠棋子 | 清爽對比 |
| 糖果馬卡龍 Candy | 粉紫＋奶油黃＋草莓紅棋子／天空藍棋子 | 可愛甜美 |
| 夜間模式 Night | 深藍黑＋螢光配色棋子（紫／青） | 保護眼睛、對比清楚 |
| 森林綠 Forest | 深綠＋卡其＋赭紅棋子／鵝黃棋子 | 自然沉穩 |

### 6.3 配色可及性原則
- 所有主題均需通過 WCAG AA 對比度檢測（文字與背景對比度 ≥ 4.5:1）。
- 玩家棋子顏色需相互區隔明顯，並額外搭配**圖示形狀差異**（如：紅方棋子內含圓點花紋、黃方棋子內含星形花紋），避免色盲玩家無法分辨雙方棋子。
- 禁止使用低對比灰配灰、螢光綠配白等不易辨識組合。

## 7. 音樂與音效規格（第 6、7 點）

### 7.1 BGM
- 提供至少 4 首**輕快鋼琴風格**背景音樂，於主選單與遊戲中依序循環播放（Playlist 模式），玩家可於設定頁切換是否隨機播放。
- 音樂需為原創或合法授權之無版權疑慮音樂素材，統一轉檔為 `.mp3`（相容性最佳）。

### 7.2 音效
- 使用**高音、清脆、短促**的音效（如：木琴／鈴鐺／氣泡感）：
  - 落子：清脆「叩」聲
  - 勝利：明亮上揚音階
  - 按鈕點擊：短促「叮」聲
  - 錯誤操作（如：欄位已滿）：輕柔提示音，不使用刺耳警報聲

### 7.3 音量放大實作方式（重要技術備註）
> 需求第 7 點提出「遊戲中的 BGM 音量都放大為原來的 10 倍」。由於瀏覽器音訊系統的音量增益若直接乘以 10 倍常會造成**嚴重破音（clipping）與對玩家聽力造成不適甚至傷害**，因此本規格書將此需求落實為以下可行方案，滿足「BGM 在遊戲中比預設更響亮、更突出」的體驗目標，同時避免音訊失真與聽力風險：

- 採用 Web Audio API 的 `GainNode` 對 BGM 音軌進行增益控制，**遊戲對戰畫面（Game Screen）中的 BGM 基準音量會比主選單提高（預設為主選單音量的等比放大，內部以線性 gain 值調整，並限制最大增益上限避免破音／削波）**。
- 設定頁提供 BGM 獨立音量滑桿（0%～200%，以視覺化「倍數」呈現給玩家：例如「1x／1.5x／2x」），系統會將使用者選擇值與內部安全上限（避免破音的技術上限）取最小值後套用，並在滑桿超過建議安全值時顯示提示文字：「音量過高可能造成破音，建議之設定範圍」。
- 遊戲一開始進入時提供「音量漸強（fade-in）」處理，避免瞬間巨大音量嚇到玩家（維護使用者聽力安全與良好體驗）。
- 音效（SFX）音量與 BGM 音量分開控制，避免落子音效被 BGM 蓋過或音量比例失衡。

## 8. 多國語系規格（第 9 點）

### 8.1 支援語言
- 繁體中文（zh-TW）— 預設語言（若瀏覽器語系判斷為中文系）
- 英文（en）
- 日文（ja）

### 8.2 實作方式
- `js/i18n/lang-*.json` 存放各語言之 key-value 文字包，例如：
  ```json
  {
    "menu.start": "開始遊戲",
    "menu.continue": "繼續遊戲",
    "menu.howto": "遊戲說明",
    "menu.settings": "設定",
    "game.yourTurn": "輪到你了",
    "game.aiTurn": "AI 思考中...",
    "result.win": "恭喜獲勝！",
    "result.lose": "再接再厲！",
    "result.draw": "平手！"
  }
  ```
- `i18n.js` 於啟動時讀取 `localStorage` 中儲存的語系設定；若無設定，則依 `navigator.language` 自動偵測並選擇最接近的支援語言，找不到對應語言時預設回退為英文。
- 所有畫面文字、按鈕、彈窗、提示訊息、遊戲說明內容皆需支援三語系切換，切換時**即時生效不需重新整理頁面**。
- 日期／數字格式依語系調整（如統計勝率呈現）。

## 9. 說明頁面規格（第 10 點）

### 9.1 內容架構
說明頁面採**分段卡片 + 圖解**方式呈現，並附有目錄錨點方便跳轉：

1. **遊戲目標**：圖解四種連線方式（橫向／直向／左斜／右斜），每種搭配一張棋盤示意 SVG 圖，連線處以高亮色框標示。
2. **基本操作**：
   - 桌機：滑鼠點擊欲落子的欄位圖示。
   - 手機：手指點擊欄位，並提供觸控操作動畫示意圖（手指點擊圖示）。
3. **勝負判定**：說明四子連線即獲勝、棋盤填滿無人連線則平手，並列出常見誤判情境（例如：斜線需連續四格）。
4. **AI 難度說明**：以可愛表情圖示對應四種難度（新手／普通／困難／專家），並簡述各難度行為差異（見第 10 章 AI 規格）。
5. **介面導覽**：標註遊戲畫面各按鈕功能（悔棋、重新開始、暫停、音量快捷鍵、回主選單）之圖示與文字說明，使用實際畫面截圖示意搭配編號標籤。
6. **常見問題（FAQ）**：例如「如何繼續上次的對局？」「可以中途更換難度嗎？」等。

### 9.2 呈現規範
- 大量使用 SVG 圖示與示意圖，避免純文字堆疊造成閱讀疲勞。
- 每個段落標題使用大字體加粗＋圖示前綴，段落間距寬鬆。
- 支援手機直式捲動閱讀，段落之間有明顯分隔線／卡片陰影區隔。
- 頂部提供「返回主選單」與語言切換快捷按鈕，方便隨時查閱不同語言版本。

## 10. AI 對戰演算法與難度設計（第 18 點）

### 10.1 難度分級

| 難度 | 演算法策略 | 行為描述 |
|---|---|---|
| 新手 Easy | 隨機落子 + 僅防守「對手下一步立即獲勝」之情況 | 大部分隨機選擇合法欄位，僅在對手即將連成四子時才會防守 |
| 普通 Normal | Minimax（搜尋深度 2～3層）+ 基礎盤面評分 | 會嘗試建立自己的連線機會，並防守明顯威脅，但不會做長遠規劃 |
| 困難 Hard | Minimax + Alpha-Beta 剪枝（搜尋深度 5～6層）+ 進階盤面評分（中央欄位加權、雙威脅偵測） | 具備一定攻防策略，能識別「雙重威脅」陷阱 |
| 專家 Expert | Minimax + Alpha-Beta 剪枝（搜尋深度 8層以上，依裝置效能動態調整）+ 開局定式資料庫 + 完整威脅評估 | 接近最佳解，善用先手優勢與開局定式，極難擊敗 |

### 10.2 演算法設計重點
- **盤面評分函式（heuristic）**：依據「連二」「連三」「中央欄位控制」「阻擋對手連線」等因子加權計算分數。
- **Alpha-Beta 剪枝**：於困難／專家難度導入，以降低搜尋節點數、提升行動裝置運算效能，避免卡頓。
- **效能保護機制**：AI 思考時間設有上限（如 1.5 秒），超過時間則採用當前最佳解，避免手機端運算過久造成畫面凍結；思考期間顯示「AI 思考中...」動畫，不阻塞 UI 執行緒（可搭配 `requestIdleCallback` 或分批運算 + `setTimeout` 讓出主執行緒）。
- **難度可於「開始遊戲」流程中選擇**，並可記錄玩家對各難度的勝率統計於設定或個人紀錄頁面（如未來擴充功能）。

## 11. 設定頁面規格（第 12 點）

### 11.1 版面配置
設定頁面採**單欄分區塊列表**排版，桌機置中卡片、手機全螢幕呈現，區塊如下：

1. **語言設定**：下拉選單或旗幟圖示按鈕（中／英／日）
2. **主題配色**：色票縮圖卡片式選擇（點選後即時預覽棋盤配色）
3. **音量設定**：
   - BGM 音量滑桿（含靜音切換圖示按鈕）
   - 音效（SFX）音量滑桿（含靜音切換圖示按鈕）
   - 滑桿採大顆圓形拖曳把手，方便手機觸控操作，並即時顯示數值百分比
4. **預設 AI 難度**：四個難度圖示卡片供選擇（作為「開始遊戲」時的預設值，仍可於每次開局前調整）
5. **其他**：震動回饋開關（行動裝置）、動畫特效開關（供低效能裝置關閉部分特效以提升流暢度）

### 11.2 視覺規範
- 每個設定項目一列，圖示＋文字說明＋操作元件（滑桿／開關／選單）三段式排版，對齊工整。
- 開關元件（Toggle）使用圓潤膠囊造型，開啟／關閉狀態顏色需高對比明確區分。
- 設定變更即時儲存於 `localStorage` 並即時套用，不需「儲存」按鈕，但提供「恢復預設值」按鈕於頁面底部。

## 12. 遊戲畫面規格（擬真、豐富、流暢）（第 13、15、17 點）

### 12.1 擬真視覺
- 棋盤採木質或大理石紋理（依主題），具備輕微立體陰影與邊框厚度感（box-shadow 模擬立體棋盤）。
- 棋子落下時具備「重力掉落」動畫（依欄位高度計算掉落時間，越高掉落時間越長，並帶有輕微彈跳緩衝效果），落地時觸發清脆音效與棋盤輕微震動效果（CSS transform 模擬）。
- 棋子造型立體圓潤，具備高光反射與陰影，呈現實體棋子質感。
- 勝利連線以動畫方式高亮四子（如：連線發光＋輕微縮放脈動），並撒落慶祝彩帶／星星等可愛特效（SVG/Canvas 粒子動畫）。

### 12.2 流暢度與效能（第 15 點）
- 所有動畫優先使用 CSS Transform / Opacity（GPU 加速），避免使用會觸發 reflow 的屬性。
- 棋盤渲染採用局部更新（僅重繪變動的格子），避免整盤重新渲染。
- AI 運算與動畫渲染分離，避免 AI 思考時造成畫面卡頓（見 10.2 效能保護機制）。
- 圖片與音訊資源皆採預先載入（preload）於載入畫面（Loading Screen）處理，避免遊戲中途出現資源延遲載入的頓卡。
- 目標效能：主流行動裝置需維持 60fps 落子與勝利動畫流暢度。

### 12.3 遊戲中介面
- 頂部資訊列：顯示雙方玩家（或玩家 vs AI）目前回合、比分、剩餘時間（若有限時模式，屬未來擴充）。
- 底部／側邊工具列（依 RWD 規則不遮擋棋盤）：悔棋、重新開始、暫停（開啟選單：繼續／回主選單／說明／設定）、音量快捷切換。
- 結算彈窗：顯示勝負結果（大字體 + 可愛插圖，如勝利煙火、落敗鼓勵貓咪插圖）、再玩一次／回主選單按鈕。

## 13. 圖示風格規範（第 17 點）

- 全站圖示統一採用**可愛圓潤插畫風格（如：圓角線條、大眼睛表情、柔和配色）**，避免過於銳利或工業風格的扁平圖示。
- AI 難度圖示採用擬人化表情設計：
  - 新手：微笑新手貓咪／小雞造型
  - 普通：認真表情的小狐狸
  - 困難：專注表情的貓頭鷹
  - 專家：戴博士帽或皇冠的神秘角色
- 功能圖示（設定齒輪、音量喇叭、說明書、返回箭頭等）皆統一線條粗細與圓角弧度，維持整體視覺一致性。
- 圖示格式統一使用 SVG，並以 `currentColor` 或 CSS 變數控制顏色，方便配合不同主題自動切換配色。

## 14. 遊戲流程與存檔規則

### 14.1 主要流程圖（文字描述）
```
主畫面
 ├─ 開始遊戲 → 選擇模式（人 vs AI / 人 vs 人）→ （若 AI）選擇難度 → 遊戲畫面
 ├─ 繼續遊戲 → 讀取存檔 → 遊戲畫面（若無存檔則按鈕disabled）
 ├─ 遊戲說明 → 說明頁面 → 返回主畫面
 └─ 設定 → 設定頁面 → 返回主畫面
```

### 14.2 存檔規則
- 每次落子後即時將目前棋盤狀態、回合數、玩家順序、難度設定寫入 `localStorage`（key: `connectfour_savegame`）。
- 若一局結束（勝利／平手），清除存檔，「繼續遊戲」按鈕恢復為不可點擊狀態。
- 悔棋功能：可回溯至上一步（雙人模式回溯 1 步；AI 模式回溯 2 步，同時撤銷玩家與 AI 的落子）。
