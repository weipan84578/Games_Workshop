## 1. 遊戲概述

### 1.1 簡介

本遊戲靈感來自 90 年代經典街機光槍遊戲（Virtua Cop、Time Crisis 系列），以**純前端技術**實作，無需後端，可直接在瀏覽器中執行。玩家透過點擊（桌面）或觸控（行動裝置）模擬射擊，在限制時間內擊倒敵人、保護人質、管理彈藥，完成各關卡挑戰。

### 1.2 核心目標

- 在倒數計時結束前取得盡可能高的分數
- 擊倒所有敵人（含 Boss）
- 避免誤傷人質
- 適時換彈，不讓彈夾耗盡

### 1.3 遊戲風格

| 項目 | 說明 |
|------|------|
| 視角 | 固定第一人稱視角（FPS 靜態場景） |
| 操作方式 | 滑鼠點擊 / 觸控點擊 |
| 場景風格 | 像素風 or 扁平卡通風（可依美術決定） |
| 遊戲長度 | 每關約 60～120 秒 |
| 難度曲線 | 由易到難，共 5 關 ＋ 無盡模式 |

## 2. 核心玩法機制

### 2.1 射擊

- 玩家點擊畫面任意位置即觸發射擊
- 每次射擊消耗 1 發子彈
- 命中判定：點擊點落在目標的碰撞範圍（hitbox）內即視為命中
- 桌面版：滑鼠左鍵射擊；右鍵或特定按鍵換彈
- 行動版：單指點擊射擊；雙指點擊或畫面底部換彈按鈕換彈

### 2.2 命中回饋

命中後依目標類型給予即時視覺與音效回饋：

| 命中目標 | 視覺效果 | 音效 |
|----------|----------|------|
| 一般敵人 | 紅色血花特效 ＋ 敵人倒下動畫 | 射擊聲 ＋ 擊中聲 |
| Boss | 受傷閃爍效果 ＋ 血條減少 | 低沉受擊音 |
| 人質 | 白色閃光 ＋ 畫面震動 | 悲鳴音效 ＋ 警示音 |
| 未命中 | 彈孔特效出現在背景牆上 | 空槍或跳彈聲 |

### 2.3 時間限制

- 每關開始時顯示倒數計時器（秒數）
- 計時器歸零時，關卡強制結束，進入結算畫面
- 特定事件（如消滅特定敵人）可觸發加時獎勵（+5 秒 / +10 秒）

### 2.4 連擊系統（Combo）

- 連續命中敵人（不誤傷人質）可觸發連擊計數器
- 連擊數越高，每次命中得分加成越高
- 誤傷人質或超過 3 秒未命中，連擊計數歸零

## 3. 畫面與頁面結構

### 3.1 頁面列表

```
主畫面 (Main Menu)
  ├── 開始遊戲 → 關卡選擇畫面
  ├── 設定畫面 (Settings)
  ├── 排行榜畫面 (Leaderboard)
  └── 說明畫面 (How to Play)

關卡選擇畫面 (Stage Select)
  └── 選擇關卡 → 過場動畫 → 遊戲畫面

遊戲畫面 (Gameplay)
  ├── HUD（血量、彈藥、分數、計時器、連擊數）
  ├── 暫停選單 (Pause Menu)
  └── 結算畫面 (Result Screen)
       ├── 過關 → 下一關 / 關卡選擇
       └── 失敗 → 重試 / 主畫面
```

### 3.2 主畫面

- 背景：動態滾動的城市夜景或警察局場景（CSS animation 實作）
- 遊戲標題：大型標題字，搭配閃爍或掃光特效
- 選單按鈕：垂直排列，字體大（建議 `font-size: clamp(1.4rem, 4vw, 2rem)`）
- 主題音樂自動播放（可靜音）
- 版本號顯示於右下角

**主畫面按鈕列表：**

| 按鈕文字 | 功能 |
|----------|------|
| 🎮 開始遊戲 | 進入關卡選擇畫面 |
| ⚙️ 設定 | 進入設定畫面 |
| 🏆 排行榜 | 查看本機最高分記錄 |
| ❓ 說明 | 顯示操作說明覆蓋層 |

### 3.3 設定畫面

- 音樂音量滑桿（0～100%）
- 音效音量滑桿（0～100%）
- 難度選擇（簡單 / 普通 / 困難）
- 語言切換（中文 / English）
- 視覺效果開關（血花特效 ON/OFF，低配裝置可關閉）
- 返回主畫面按鈕
- **字體大小**：所有設定項目 label `font-size` 不小於 `1.2rem`，滑桿高度不小於 `44px`（符合行動端點擊標準）

### 3.4 遊戲畫面 HUD

```

│  ❤️ x 3        STAGE 1        ⏱ 00:45    💥 x3 COMBO │
```

| HUD 元件 | 說明 |

## 4. 計分系統

### 4.1 基礎分數

| 事件 | 分數 |
|------|------|
| 命中一般敵人 | +100 分 |
| 命中敵人要害（頭部）| +300 分 |
| 消滅 Boss | +1000 分 |
| 時間獎勵（剩餘每 1 秒）| +50 分 |
| 全關無傷通過 | +500 分 |
| 全關無誤傷人質 | +300 分 |

### 4.2 扣分規則

| 事件 | 扣分 |
|------|------|
| 誤傷人質 | -500 分（分數不低於 0） |
| 生命值歸零 | 關卡失敗，不計分 |
| 彈夾耗盡後仍嘗試射擊 | -50 分（空槍懲罰） |

### 4.3 連擊加成

| 連擊數 | 分數倍率 |
|--------|---------|
| 1～4 連擊 | ×1.0 |
| 5～9 連擊 | ×1.5 |
| 10～19 連擊 | ×2.0 |
| 20 連擊以上 | ×3.0 |

### 4.4 本機排行榜

- 使用 `localStorage` 儲存前 10 筆最高分
- 記錄：玩家名稱（可自訂，預設 "PLAYER"）、分數、關卡達成、日期
- 結算畫面顯示是否達成新紀錄

## 5. 彈藥與換彈機制

### 5.1 彈夾容量

| 難度 | 彈夾容量 | 備用彈藥上限 |
|------|----------|-------------|
| 簡單 | 無限 | — |
| 普通 | 12 發 | 無限換彈 |
| 困難 | 8 發 | 無限換彈，換彈時間較長 |

### 5.2 換彈觸發方式

| 平台 | 操作 |
|------|------|
| 桌面版 | 按 `R` 鍵 / 滑鼠右鍵 |
| 行動版 | 點擊畫面底部「換彈」按鈕 / 雙指點擊 |
| 彈夾耗盡 | 自動觸發換彈（含動畫與音效） |

### 5.3 換彈動畫與時間

- 換彈期間畫面左下角顯示換彈進度條
- 進度條填滿後換彈完成，可繼續射擊
- 換彈期間仍可移動畫面（若有）但無法射擊

| 難度 | 換彈時間 |
|------|---------|
| 簡單 | 1.0 秒 |
| 普通 | 1.5 秒 |
| 困難 | 2.0 秒 |

### 5.4 換彈視覺提示

- 子彈剩餘 3 發以下時，彈藥顯示框變紅色並閃爍
- 換彈期間顯示旋轉的子彈圖示動畫
- 換彈完成後彈藥框恢復正常顏色，伴隨「咔嚓」音效

## 6. 敵人與人質系統

### 6.1 敵人類型

| 類型 | 說明 | 生命值 | 出現時間 | 特性 |
|------|------|--------|----------|------|
| 普通敵人 | 標準武裝歹徒 | 1 發 | 1～2 秒 | 基礎行為 |
| 盾牌敵人 | 持盾，需命中露出部位 | 2 發 | 2 秒 | 遮擋彈道 |
| 快速敵人 | 快速閃現，停留時間短 | 1 發 | 0.5 秒 | 快速進出 |
| 遠距敵人 | 體型較小 | 1 發 | 2 秒 | 目標小 |
| Boss | 關卡結尾出現 | 多段 | 不消失 | 有攻擊模式 |

### 6.2 人質類型

| 類型 | 說明 | 誤傷後果 |
|------|------|---------|
| 一般人質 | 平民，無武裝 | -500 分 ＋ 警示效果 |
| 警察人質 | 穿制服人物 | -500 分 ＋ 警示效果 |
| 特殊人質 | 標記「VIP」 | -1000 分 ＋ 生命值 -1 |

### 6.3 敵人行為邏輯

1. **出現（Spawn）**：敵人從預設位置「彈出」，伴隨出現音效
2. **停留（Active）**：維持在螢幕上一段固定時間，等待玩家射擊
3. **攻擊（Attack）**：若玩家在停留時間內未射擊，敵人執行攻擊，玩家扣血
4. **消失（Despawn）**：攻擊後收回，或被擊倒後播放死亡動畫

```
[Spawn] → [Active 停留 N 秒] → [被擊倒] → [Death 動畫] → 結束
                              ↘ [未被擊倒] → [Attack] → 玩家扣血 → [Despawn]
```

### 6.4 敵人出現位置

場景劃分為 **3 × 3 的九宮格區域**，每個區域可獨立觸發敵人出現，確保玩家需要注意整個畫面。

## 7. 關卡設計

### 7.1 關卡一覽

| 關卡 | 場景 | 時間限制 | 主題敵人 | 特殊機制 |
|------|------|----------|----------|---------|
| Stage 1 | 倉庫 | 90 秒 | 普通敵人 | 教學關，敵人出現提示明顯 |
| Stage 2 | 城市街道 | 80 秒 | 普通 ＋ 快速敵人 | 人質開始大量出現 |
| Stage 3 | 銀行大廳 | 70 秒 | 盾牌敵人為主 | 換彈時機更重要 |
| Stage 4 | 停車場 | 70 秒 | 混合敵人 | 夜晚場景，可見度降低 |
| Stage 5 | 頂樓 | 60 秒 + Boss | 遠距敵人 ＋ Boss | 最終 Boss 戰 |
| 無盡模式 | 隨機場景 | 無 | 隨機混合 | 關卡每輪難度遞增 |

### 7.2 關卡難度遞增規則

- 每關增加敵人同時出現數量（最多 5 個）
- 縮短敵人停留時間
- 增加人質與敵人混出頻率
- 後期關卡加入複合動作敵人

### 7.3 Boss 戰規格（Stage 5）

- Boss 分 3 階段，每階段血量耗盡進入下一階段
- 各階段更換攻擊模式與弱點位置
- Boss 攻擊時畫面震動並播放警示音
- 擊敗 Boss 後播放勝利演出

## 8. 音樂與音效系統

### 8.1 技術實作

- 使用 **Web Audio API** 作為核心音訊引擎
- 搭配 **Howler.js**（CDN 引入）管理音訊播放、淡入淡出、循環
- 所有音效以 **Base64 編碼**內嵌於 JS 或以相對路徑載入
- 首次使用者互動（點擊/觸控）後才啟動音訊播放（遵守瀏覽器 Autoplay Policy）

### 8.2 背景音樂列表

| 音樂 ID | 使用場景 | 風格 | 循環 |
|---------|---------|------|------|
| `bgm_menu` | 主畫面、設定、排行榜 | 緊張電子節拍 | 是 |
| `bgm_stage1` | Stage 1 | 輕快動作風 | 是 |
| `bgm_stage2` | Stage 2 | 城市節拍 | 是 |
| `bgm_stage3` | Stage 3 | 緊張爵士 | 是 |
| `bgm_stage4` | Stage 4 | 暗黑電子 | 是 |
| `bgm_stage5` | Stage 5 + Boss | 史詩管絃 ＋ 電子 | 是 |
| `bgm_endless` | 無盡模式 | 快節奏 Loop | 是 |
| `bgm_result` | 結算畫面 | 短樂段 | 否 |

> **建議音源**：使用 OpenGameArt.org 或 Freesound.org 的 CC0 授權音樂，或以 Web Audio API 程序生成 8-bit 風格音樂。

### 8.3 音效列表

| 音效 ID | 觸發時機 | 描述 |
|---------|---------|------|
| `sfx_gunshot` | 每次射擊 | 單聲道槍響，短促有力 |
| `sfx_hit_enemy` | 命中敵人 | 擊中肉體的鈍響 |
| `sfx_hit_headshot` | 命中要害 | 特別清脆的爆頭音 |
| `sfx_hit_hostage` | 誤傷人質 | 悲鳴 ＋ 低沉警示音 |
| `sfx_miss` | 未命中（彈孔） | 低沉跳彈聲 |
| `sfx_empty_gun` | 彈夾空了 | 扣板機空槍聲 |
| `sfx_reload_start` | 開始換彈 | 彈夾退出聲 |
| `sfx_reload_done` | 換彈完成 | 彈夾裝入聲（咔嚓） |
| `sfx_enemy_spawn` | 敵人出現 | 快速移動聲 |
| `sfx_enemy_attack` | 敵人攻擊玩家 | 衝擊聲 ＋ 玩家受傷音 |
| `sfx_enemy_die` | 敵人死亡 | 倒地聲 |
| `sfx_boss_hit` | Boss 受傷 | 低沉金屬受擊音 |
| `sfx_boss_phase` | Boss 進入下一階段 | 戲劇性音效 |
| `sfx_combo` | 達到特定連擊數 | 語音提示（如 "COMBO!"） |
| `sfx_time_warning` | 倒數最後 10 秒 | 急促計時嗶聲 |
| `sfx_stage_clear` | 過關 | 勝利音效 |
| `sfx_game_over` | 失敗 | 遊戲結束音效 |
| `sfx_ui_click` | UI 按鈕點擊 | 短促 UI 音 |
| `sfx_ui_hover` | 桌面版按鈕 hover | 輕微電子音 |

### 8.4 音量控制

```javascript
// 音量層級架構
masterVolume         // 全域音量（0.0 ～ 1.0）
  ├── musicVolume    // 音樂音量，獨立控制
  └── sfxVolume      // 音效音量，獨立控制
```

- 設定畫面提供兩組滑桿分別控制音樂與音效音量
- 音量設定儲存至 `localStorage`，下次遊玩時自動套用
- 靜音按鈕（主畫面右上角）可一鍵切換全域靜音

- 禁用狀態需視覺區隔（`opacity: 0.4`）

## 10. 行動裝置適配規範

### 10.1 RWD 斷點

```css
/* 手機直向 */
@media (max-width: 480px) { ... }

/* 手機橫向 / 小型平板 */
@media (min-width: 481px) and (max-width: 768px) { ... }

/* 平板 */
@media (min-width: 769px) and (max-width: 1024px) { ... }

/* 桌面 */
@media (min-width: 1025px) { ... }
```

### 10.2 觸控操作設計

- 遊戲主體區域綁定 `touchstart` 事件觸發射擊（非 `click`，減少延遲）
- 禁用 `touch-action: none` 於遊戲區域，防止誤觸捲動
- 換彈按鈕固定於畫面右下角，大小至少 `64 × 64 px`
- 暫停按鈕固定於畫面左上角
- 防止遊戲過程中觸發系統返回手勢（`History API` 鎖定）

### 10.3 行動版 HUD 佈局調整

```
手機直向（portrait）:
┌────────────────────┐
│ [暫停] STAGE 1  ⏱ │  ← 頂部 HUD 列（字體大）
│ ❤️❤️❤️   💥 x5   │
│                    │

│                    │

└────────────────────┘
```

### 10.4 效能優化（行動裝置）

- 使用 CSS `will-change: transform` 於頻繁動畫元素
- 特效（血花、彈孔）使用物件池（Object Pooling）複用 DOM 元素
- 場景背景使用靜態圖片，不使用複雜 CSS 動畫
- 減少 `box-shadow` 與 `filter` 的使用（行動 GPU 成本高）
- 音訊檔案使用 MP3 格式（最佳相容性），WebM 作備援

### 10.5 螢幕方向

- **建議橫向（landscape）**：遊戲進入時顯示提示圖示，引導用戶旋轉裝置
- 橫向遊玩時，HUD 分配至左右側邊欄
- 仍支援直向遊玩，但場景適當壓縮

```javascript
function gameLoop(timestamp) {
```

### 11.4 輸入系統

```javascript
// 統一輸入處理，支援滑鼠與觸控
class InputHandler {
  constructor(canvas) {
    canvas.addEventListener('mousedown', this.handleShoot.bind(this));
    canvas.addEventListener('touchstart', this.handleShoot.bind(this), { passive: false });
    document.addEventListener('keydown', this.handleKey.bind(this));
  }

  handleShoot(e) {
    e.preventDefault();
    const pos = this.getPosition(e);  // 統一取得座標
    GameState.player.shoot(pos.x, pos.y);
  }
}
```
