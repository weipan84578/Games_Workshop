## 1. 專案總覽

### 1.1 專案簡介
一款可在瀏覽器直接開啟的擬真骰寶（Sic Bo / Tai Sai）網頁遊戲。使用者以虛擬籌碼在賭桌上下注，透過搖骰結果判定輸贏並結算籌碼。遊戲需具備完整選單系統、多國語系、豐富音效與流暢動畫，且**全程不需連網、不需伺服器、不需任何打包建置流程**。

### 1.2 核心設計原則
| 原則 | 說明 |
|---|---|
| 零建置 | 所有程式碼皆為原生 HTML / CSS / JavaScript（ES6+），無 Webpack / Vite / npm 依賴 |
| 開箱即玩 | 雙擊 `index.html` 直接於瀏覽器開啟，無需 `http-server`、`live-server` 等指令 |
| 響應式優先 | 同一套程式碼需同時服務桌機與行動裝置，且行動裝置操作區不可遮擋骰盤 |
| 高可讀性 | 全站字體加大加粗，資訊層級清楚 |
| 沉浸式體感 | 賭桌質感、骰盅骰子皆採擬真 3D 質感呈現 |
| 模組化 | CSS／JS 依功能拆分至獨立資料夾與檔案，`index.html` 純粹以 `<link>` `<script>` 引入 |

## 2. 技術規範與限制

### 2.1 允許使用的技術
- **HTML5**：語意化標籤（`<header>`, `<main>`, `<section>`, `<dialog>` 等）
- **CSS3**：Flexbox、Grid、CSS 變數（自訂屬性）、`clamp()` 做流體字體、CSS 動畫／transition、`@media` 查詢
- **原生 JavaScript（ES6+ Module 或傳統 script 皆可，建議傳統 `<script src>` 以避免部分瀏覽器對 `file://` 開啟 ES Module 的 CORS 限制）**
- **Web Audio API / HTMLAudioElement**：音效與 BGM 播放與音量控制
- **LocalStorage**：儲存進度、語言、音量、配色主題等設定

### 2.2 禁止使用
- ❌ 任何需要 `npm install`、`build`、`bundle` 的框架（React/Vue 需 build 版本不算純前端，故不使用）
- ❌ 任何需要後端 API、資料庫、伺服器渲染的技術
- ❌ 需要網路 CDN 才能運作的核心功能（字體、圖示可有 CDN 備援，但需提供離線 fallback）

> ⚠️ **重要技術注意事項**：由於遊戲以 `file://` 協定直接開啟（雙擊 index.html），JavaScript 需避免使用 `fetch()` 讀取本地 JSON（多數瀏覽器會因 CORS 政策封鎖 `file://` 下的 fetch）。因此：
> - 多國語系字典、下注賠率表等資料，一律以 **JS 物件（`const dict = {...}`）** 的形式寫在 `.js` 檔中並用 `<script>` 引入，**不使用外部 `.json` + fetch**。
> - 音樂／音效／圖片則可直接以相對路徑於 `<audio>` `<img>` 或 CSS `url()` 引入，此類資源讀取不受 CORS 限制。

### 2.3 瀏覽器支援
- Chrome / Edge 最近 3 個大版本
- Safari 16+（含 iOS Safari）
- Firefox 最近 3 個大版本

## 4. RWD 響應式設計規範

### 4.1 斷點定義
| 裝置類型 | 寬度範圍 | 版面策略 |
|---|---|---|
| 手機直向 | ≤ 480px | 骰盤全螢幕置中，下注區可左右滑動；操作按鈕收合於底部安全區 |
| 手機橫向／小平板 | 481px–767px | 賭桌與下注區並排縮放顯示 |
| 平板 | 768px–1024px | 賭桌置中，籌碼列固定於畫面下方 1/5 高度 |
| 桌機 | ≥ 1025px | 完整賭桌 + 側邊資訊欄（餘額、歷史紀錄、下注統計）同時顯示 |

### 4.2 行動裝置關鍵規則（對應需求 2 / 8）
- 所有**操作按鈕（下注確認、清除下注、開始搖骰、返回選單）**一律固定於畫面**最下方安全區（`env(safe-area-inset-bottom)`）**，並使用**半透明浮動列（floating action bar）**，不得覆蓋骰盤與下注區主要視覺。
- 骰盤／骰子動畫顯示區域使用 `min-height` 保留，操作列以 `position: fixed` + `z-index` 分層，並在骰子動畫播放時**自動降低操作列透明度**，避免視覺干擾。
- 下注面板在手機模式下改為**可橫向滑動的卡片列（horizontal scroll snap）**，避免直式塞爆畫面。
- 使用 `100dvh`（動態視窗高度）取代 `100vh`，避免手機瀏覽器網址列造成版面跳動或遮擋。
- 所有可點擊區域最小 **44×44px**（符合行動裝置觸控標準）。

### 4.3 版面彈性技巧
- 字體、間距使用 `clamp(最小值, 流體值, 最大值)`，例如：`font-size: clamp(16px, 2.5vw, 22px)`
- 賭桌容器使用 `aspect-ratio` 鎖定比例並依容器縮放（`transform: scale()` 搭配 `container query` 概念手動計算）
- 橫向 / 直向切換時觸發 `resize` / `orientationchange` 事件重新排版籌碼與下注格子座標

## 5. 視覺與 UI 設計規範

### 5.1 字體規範（對應需求 3）
| 用途 | 桌機字級 | 手機字級 | 字重 |
|---|---|---|---|
| 主標題（LOGO/頁面標題） | 40–56px | 28–34px | 800 (ExtraBold) |
| 次標題（區塊名稱） | 24–28px | 18–20px | 700 (Bold) |
| 內文／說明文字 | 18–20px | 16–17px | 500 (Medium) |
| 按鈕文字 | 20–22px | 18px | 700 (Bold) |
| 數字（餘額/賠率/骰子點數） | 26–32px | 20–24px | 800，使用等寬數字字體特性 `font-variant-numeric: tabular-nums` |

- 全站**不使用低於 16px 的字級**，確保行動裝置閱讀清晰。
- 中文使用「思源黑體 / Noto Sans TC」、日文使用「Noto Sans JP」、英文使用「Poppins / Nunito」等具現代感且清晰易讀的字體，並提供本地 `assets/fonts` 離線字型作為 CDN 失敗時 fallback。
- 文字一律搭配足夠對比度背景（文字與背景對比度 ≥ 4.5:1），並在深色與淺色主題下分別測試可讀性，避免需求 11 所提「看不見」的情況。

### 5.2 配色主題（對應需求 3 / 11）
提供 **5 種可切換配色主題**（於設定頁選擇），每個主題皆定義以下 CSS 變數：

```css
:root[data-theme="classic-red"] {
  --color-primary: #C0392B;
  --color-secondary: #D4AF37; /* 金色 */
  --color-bg-table: #7A1F1F;
  --color-bg-page: #1A1A1A;
  --color-text-main: #FFF8E7;
  --color-text-secondary: #E0C68C;
  --color-success: #2ECC71;
  --color-danger: #E74C3C;
  --color-warning: #F1C40F;
}
```

各主題（經典中式紅金、翡翠綠寶石、皇家藍紫、暗夜黑金、櫻花粉）皆須確保：
- 主要文字與背景對比度合格
- 強調色（勝利/失敗/警告）在任何主題下都清晰可辨，不使用相近色相造成混淆
- 骰子、籌碼顏色不隨主題改變（維持擬真紅／白骰子與寫實籌碼配色），僅背景、按鈕、UI 框架跟隨主題色

### 5.3 視覺豐富度（對應需求 11 / 13）
- 賭桌採用**綠色氈布材質貼圖 + 木質桌框陰影**，呈現真實賭場質感
- 骰盅／骰子使用 CSS 3D Transform（`rotateX/Y/Z` + `perspective`）呈現立體滾動效果，骰子加上動態陰影與高光
- 籌碼採用堆疊層次陰影（`box-shadow` 多層堆疊），下注時有「輕拋入位」的緩動動畫
- 中獎時觸發**金幣灑落／閃光粒子特效**（純 CSS/Canvas 輕量特效，不使用重型函式庫）
- 頁面背景使用微噪點紋理（noise texture）與柔光暈染，避免大面積色塊顯得單調廉價

| 設定 | 開啟設定頁（見第 12 節） |

## 7. 遊戲主畫面規格

### 7.1 畫面區塊
1. **頂部資訊列**：目前餘額、目前主題語系縮寫、回合數、返回主選單按鈕
2. **中央骰盅區**：搖骰動畫呈現區，開骰後顯示三顆骰子最終點數與總和
3. **下注面板**：完整骰寶下注區域（見第 8 節），依裝置類型呈現網格（桌機）或橫向卡片（手機）
4. **底部操作列（浮動）**：籌碼面額選擇、確認下注、清除下注、開始搖骰
5. **側邊/浮動歷史紀錄**：近 10 局開骰結果小圖示列（桌機常駐側欄，手機可收合抽屜）

### 7.2 遊戲流程
```
下注階段（可自由下注多個區域，倒數計時可選）
   ↓
鎖注（Lock Bets）→ 播放搖骰動畫（骰盅晃動 2–3 秒）
   ↓
開骰結果揭曉（骰子逐一定格顯示點數，伴隨音效）
   ↓
結算：命中區域高亮 + 派彩動畫 + 餘額更新
   ↓
顯示「再玩一局」/「返回主選單」選項，或自動進入下一輪下注階段
```

## 8. 骰寶遊戲規則與下注邏輯

### 8.1 基本規則
使用三顆六面骰子搖骰，玩家於開骰前在下注區下注，開骰後依骰子點數組合結算輸贏與賠率。

### 8.2 完整下注類型與賠率表

| 下注類型 | 說明 | 賠率（賠付：本金） |
|---|---|---|
| 大（Big） | 總和 11–17（豹子不算） | 1:1 |
| 小（Small） | 總和 4–10（豹子不算） | 1:1 |
| 單（Odd） | 總和為奇數（豹子不算） | 1:1 |
| 雙（Even） | 總和為偶數（豹子不算） | 1:1 |
| 指定總和（4 或 17） | 三骰總和精準命中 | 60:1 |
| 指定總和（5 或 16） | | 30:1 |
| 指定總和（6 或 15） | | 17:1 |
| 指定總和（7 或 14） | | 12:1 |
| 指定總和（8 或 13） | | 8:1 |
| 指定總和（9,10,11,12） | | 6:1 |
| 單骰（Single Number） | 押 1 顆指定點數，出現 1 顆賠 1 倍，2 顆賠 2 倍，3 顆（豹子）賠 3 倍 | 1/2/3 倍階梯賠率 |
| 雙骰組合（Specific Double） | 押任兩顆同點數（如 2+2） | 10:1 |
| 三軍／全圍（Any Triple） | 三顆同點數（任意數字） | 30:1 |
| 指定豹子（Specific Triple） | 押特定點數三顆全同（如 6-6-6） | 180:1 |
| 兩顆骰組合（Combination，如 1+2） | 任兩顆骰組合出指定兩數字 | 6:1 |

> 賠率表資料以 `js/core/bet-types-data.js` 統一維護，UI 顯示與賠付計算皆讀取同一資料源，避免前後端（此處指顯示層與邏輯層）數值不一致。

### 8.3 特殊規則
- 出現**任意豹子（三顆同點）**時，「大」「小」「單」「雙」一律視為**輸**（莊家通殺），此為骰寶標準規則，需於說明頁清楚圖示標註。
- 單骰下注採**階梯式賠付**：命中 1 顆賠 1 倍、2 顆賠 2 倍、3 顆賠 3 倍本金（外加退還本金）。

### 8.4 下注互動邏輯
- 玩家先選擇籌碼面額（如 10 / 50 / 100 / 500 / 1000），再點擊下注區域疊加籌碼
- 支援「清除全部下注」「取消最後一注」「重複上局下注」快捷按鈕
- 下注總額即時顯示於頂部資訊列，超出餘額時該下注區域按鈕自動禁用並提示

## 9. 音效與音樂規格

### 9.1 BGM 規範（對應需求 6 / 7）
- 主選單與遊戲中皆播放**鋼琴演奏的輕快旋律**（3 首以上輪播，避免單曲循環聽膩）
- BGM 音量在**遊戲進行中（進入遊戲主畫面後）** 相較主選單基準音量**放大為原本的 10 倍**：
  - 實作方式：`audio-manager.js` 定義 `BASE_BGM_VOLUME`（主選單使用），進入遊戲主畫面時套用 `GAME_BGM_VOLUME = Math.min(BASE_BGM_VOLUME * 10, 1.0)`，並以 `HTMLAudioElement.volume`（範圍 0–1）或 Web Audio `GainNode.gain` 實作放大。
  - 因瀏覽器音量上限為 1.0，若 `BASE_BGM_VOLUME` 設定過高導致 ×10 超過上限，程式自動 clamp 至 1.0 並於 console 記錄提示，避免破音爆音；建議 `BASE_BGM_VOLUME` 預設為 **0.08**，遊戲中即放大至 **0.8**，確保音量顯著提升但不失真。
  - 若使用 Web Audio API `GainNode`，可透過 `gain.value` 突破 1.0 上限做真正的「10 倍增益」，但需搭配 `DynamicsCompressorNode` 防止破音，此為**進階實作建議**。
- BGM 需可於設定頁單獨調整（獨立於音效音量），並記憶使用者最後設定值於 LocalStorage。

### 9.2 音效規範（對應需求 6）
所有互動音效統一採用**高音、清脆、短促**的音色特性（類似鈴鐺、水晶敲擊、清脆鐘聲）：

| 事件 | 音效特性 |
|---|---|
| 按鈕點擊 | 極短促清脆「叮」聲 |
| 籌碼下注 | 清脆碰撞聲（類似硬幣輕觸） |
| 搖骰晃動 | 清亮骰子碰撞聲（連續短音） |
| 骰子落定 | 清脆定格「噠」聲 |
| 小獎中獎 | 高音音階上揚鈴聲 |
| 大獎中獎 | 華麗高音琶音＋鈴鐺齊鳴 |
| 未中獎 | 輕柔低調提示音（仍維持不刺耳） |
| 頁面切換 | 輕快「唰」聲 |

### 9.3 音訊架構
- `audio-manager.js` 提供統一 API：`playBGM(id)`、`stopBGM()`、`playSFX(id)`、`setBGMVolume(val)`、`setSFXVolume(val)`
- 使用者可於設定頁**分別**調整 BGM／音效音量，並可獨立靜音
- 行動裝置需處理瀏覽器「使用者互動前禁止自動播放」限制：於主畫面第一次點擊任意按鈕時觸發 `AudioContext.resume()` 並開始播放 BGM

## 10. 多國語系規格（i18n）

### 10.1 支援語言（對應需求 9）
- 🇹🇼 繁體中文（預設）
- 🇺🇸 英文
- 🇯🇵 日文

### 10.2 實作方式
- `js/i18n/lang-*.js` 各自匯出語言字典物件，例如：

```js
// lang-zh-TW.js
const LANG_ZH_TW = {
  menu: { start: "開始遊戲", continue: "繼續遊戲", howToPlay: "遊戲說明", settings: "設定" },
  game: { balance: "餘額", placeBet: "確認下注", clearBet: "清除下注", roll: "搖骰" },
  betTypes: { big: "大", small: "小", odd: "單", even: "雙", anyTriple: "全圍" },
  // ...涵蓋所有 UI 文字、規則說明文字、彈窗訊息
};
```

- `i18n-manager.js` 提供 `t(key)` 取字函式，所有畫面文字**一律透過此函式取得**，禁止在 HTML／JS 中寫死任何顯示字串。
- 語言切換即時生效（不需重新整理頁面），並將選擇語言存入 LocalStorage，下次開啟自動套用。
- 主畫面右上角提供語言切換角標（國旗圖示或 ZH / EN / JA 文字切換鈕）。

### 10.3 涵蓋範圍
多語系需完整涵蓋：主畫面、遊戲主畫面所有文字與按鈕、下注類型名稱、說明頁全部內容、設定頁選項文字、彈窗與提示訊息、錯誤訊息。

## 11. 說明頁面規格

### 11.1 內容架構（對應需求 10）
說明頁採用**分區塊、圖文並茂**的結構，非單純長文字堆疊：

1. **遊戲簡介**：一段簡短文字＋骰盅示意圖
2. **基本操作教學**（步驟圖示 Step 1 → 2 → 3 → 4）
   - 圖示化步驟：選籌碼 → 點下注區 → 確認下注 → 觀看開骰
3. **下注區域地圖**：以視覺化賭桌示意圖（`docs-assets/bet-area-map.svg`），標註每個下注區域對應名稱與位置，滑鼠 hover／點擊可彈出該區域詳細賠率說明
4. **完整賠率表**：以表格＋圖示（骰子點數示意）呈現第 8.2 節所有下注類型，並依「單骰／組合／總和／大小單雙／豹子」分類分頁籤（Tab）呈現，避免長表格難以閱讀
5. **特殊規則說明**：豹子通殺規則以圖解（三顆同色骰子 + 大叉叉圖示）清楚呈現
6. **常見問題（FAQ）**：手風琴（Accordion）收合式問答

### 11.2 版面要求
- 使用 Tab 分頁 + Accordion 收合，避免單頁資訊過載
- 每個下注類型皆搭配**小型骰子點數示意圖示**（而非純文字），提升可讀性
- 內容可捲動，但頂部固定分類導覽列方便快速跳轉
- 字體遵循第 5.1 節大字體規範，行距 1.6 以上確保長文閱讀舒適
- 支援三語系切換（見第 10 節）

## 14. 動畫與效能規範

### 14.1 流暢度要求（對應需求 15）
- 所有動畫（骰子滾動、籌碼移動、彈窗淡入淡出）目標 **60 FPS**，優先使用 CSS `transform` / `opacity` 做動畫（觸發 GPU 合成層），避免使用會觸發 reflow 的屬性（如 `top/left/width` 逐格動畫）
- 骰子 3D 滾動動畫使用 `will-change: transform` 預先告知瀏覽器優化
- 大量籌碼堆疊時採用**元素重用（Object Pooling）**而非重複 `createElement`，避免記憶體暴增
- 圖片資源使用適當壓縮（WebP 優先，PNG 備援），避免大圖拖慢行動裝置載入
- 避免同時播放過多音效導致音訊延遲，音效播放採用**音效池（Audio Pool）**預先建立多個 `<audio>` 實例輪替播放

### 14.2 降級策略
- 偵測低效能裝置（可用簡單 FPS 監測或 `navigator.hardwareConcurrency`）時，自動降低粒子特效數量與骰子陰影細節，確保操作依然流暢
