## 1. 專案總覽

本專案為一款**純前端**、**離線可執行**的飛鏢（Darts）小遊戲。使用者僅需**直接雙擊 `index.html`** 即可開啟並完整遊玩，不需要：

- 任何 Node.js / npm / yarn 安裝
- 任何本地伺服器（如 `live-server`、`http-server`）
- 任何打包建置流程（Webpack / Vite / Parcel 等）
- 任何後端 API 或資料庫

所有資源（圖片、音效、字型、語言檔）皆以相對路徑載入，並透過標準 `<link>`、`<script>` 標籤於 `index.html` 中引入，確保以 `file://` 協定開啟時仍可正常運作（音效與圖片需避免使用會被瀏覽器 CORS 政策擋下的載入方式，細節見第 16 章）。

## 2. 技術限制與執行方式

| 項目 | 規格 |
|---|---|
| 開發語言 | HTML5 + CSS3 + Vanilla JavaScript（ES6+，不依賴框架） |
| 執行方式 | 雙擊 `index.html` 直接於瀏覽器開啟 |
| 建置流程 | 無，禁止使用任何 bundler |
| 外部套件 | 不使用需要安裝的 npm 套件；若需第三方函式庫（如音效解碼），一律採「本地檔案」方式放入 `libs/` 資料夾，不使用 CDN 連結（避免離線無法遊玩） |
| 瀏覽器支援 | Chrome、Edge、Safari、Firefox 最新兩個大版本；iOS Safari / Android Chrome |
| 資料儲存 | 使用 `localStorage` 儲存進度、設定、語言、音量等資訊，不使用任何雲端服務 |
| 檔案編碼 | 全站 UTF-8 |

## 4. 響應式設計（RWD）規格

### 4.1 斷點定義

| 裝置類型 | 寬度範圍 | 佈局重點 |
|---|---|---|
| 手機直向 | ≤ 480px | 單欄佈局，靶面置中縮放，操作按鍵移至畫面下方安全區 |
| 手機橫向 / 小平板 | 481px–768px | 靶面與資訊面板左右並排，控制鍵置於左右兩側 |
| 平板 | 769px–1024px | 靶面加大，側邊資訊面板固定顯示 |
| 桌機 | ≥ 1025px | 完整佈局，靶面、記分板、玩家列表三欄式呈現 |

### 4.2 核心原則

- 所有尺寸單位優先使用 `vw`、`vh`、`clamp()`、`%`，避免寫死 `px` 造成裝置適配問題。
- 遊戲主視覺（靶面）永遠置中，並以 `aspect-ratio: 1 / 1` 鎖定為正方形，確保縮放不變形。
- 行動裝置的虛擬操作按鍵（瞄準、投擲、確認）採用**懸浮半透明圓形按鈕**，固定於畫面下方或左右邊緣，並保留至少 **20% 靶面可視安全區**，確保按鍵不遮擋靶心與飛鏢飛行路徑。
- 螢幕方向偵測：若手機為直向，自動提示「建議橫向遊玩以獲得最佳體驗」但直向仍可完整操作。
- 使用 `ResizeObserver` 監聽容器尺寸變化，動態重新計算靶面命中座標比例，避免橫向/直向切換後命中判定失準。

## 5. 字體與配色系統

### 5.1 字體規格

- 全站字體大小以**大字體、高辨識度**為原則，於 `typography.css` 統一定義字級變數：

| 用途 | 字級變數 | 參考大小（桌機） | 參考大小（手機） |
|---|---|---|---|
| 主標題（Logo/標題） | `--font-size-hero` | 64px | 40px |
| 頁面標題 | `--font-size-h1` | 40px | 28px |
| 區塊標題 | `--font-size-h2` | 30px | 22px |
| 按鈕文字 | `--font-size-button` | 24px | 20px |
| 內文說明 | `--font-size-body` | 20px | 18px |
| 分數顯示（大數字） | `--font-size-score` | 56px | 36px |
| 輔助提示文字 | `--font-size-caption` | 16px | 14px |

- 字重統一使用 `600`–`800`，避免細字重在小螢幕上模糊不清。
- 中文採用圓潤易讀字型（如思源黑體 / Noto Sans TC），日文採用 Noto Sans JP，英文採用具個性的無襯線字型（如 Poppins / Baloo 2），三者行高與字距需視覺對齊，避免切換語言時版面跳動。
- 分數、計時等關鍵數字使用等寬數字字體，避免跳動時版面抖動。

### 5.2 配色系統

- 提供 **4 組主題配色**（於 `themes/` 資料夾對應），使用者可於設定頁自由切換：

| 主題名稱 | 主色調 | 特色 |
|---|---|---|
| 經典酒吧（預設） | 深木紋棕 + 酒紅 + 金色點綴 | 擬真飛鏢酒吧場景 |
| 霓虹電競 | 深紫黑底 + 螢光藍/粉 | 電競感強烈對比 |
| 日系粉彩 | 米白 + 粉藕色 + 淺綠 | 柔和清新 |
| 深色護眼 | 深灰藍 + 低飽和橘 | 夜間長時間遊玩護眼 |

- 所有配色皆需通過**對比度檢查（WCAG AA 以上，對比比例 ≥ 4.5:1）**，避免文字與背景同色系造成「看不見」的狀況。
- 靶面本身固定使用真實飛鏢配色（黑/白/紅/綠相間），不受主題切換影響，以維持遊戲辨識度；僅背景、UI 面板、按鈕隨主題變化。
- 所有互動元件（按鈕、滑桿、輸入框）需有清楚的 `hover`、`active`、`disabled` 三態視覺差異。

## 7. 遊戲畫面規格

### 7.1 版面配置

- **桌機**：左側為玩家列表與回合資訊，中央為靶面主視覺，右側為目前分數/歷史投擲記錄。

## 8. 飛鏢投擲互動規格

### 8.1 桌機操作

- 滑鼠移動 = 瞄準準心移動（準心以十字或飛鏢剪影顯示於靶面上）。
- 按住左鍵並向後拖曳 = 蓄力（顯示力道拉弓式 UI 條）。
- 放開左鍵 = 投擲，飛鏢依「準心位置 + 力道 + 些微隨機浮動誤差」計算最終落點。

### 8.2 觸控操作

- 手指觸碰 = 準心移動。
- 手指向下滑動蓄力（模擬手臂後拉動作），放開手指 = 投擲。
- 力道 UI 條與桌機共用同一組元件，僅輸入方式不同。

### 8.3 命中判定

- 使用極座標運算：將落點座標換算為「角度（決定計分區塊 1–20）」與「半徑（決定 Single/Double/Triple/Bullseye/Outer Bull/Miss）」。
- 命中後觸發：命中音效、靶面局部震動動畫、飛鏢插入靶面的擬真插入動畫（含些微晃動物理效果後靜止）。

## 9. 遊戲規則與模式

提供三種標準飛鏢玩法，於「開始遊戲」時可選擇：

### 9.1 501 / 301（Zero Out）

- 每位玩家初始分數為 501 或 301，每輪 3 鏢，命中分數自總分扣除。
- 最終須「精準歸零」，且最後一鏢必須以 **Double（雙倍環）或 Bullseye** 收尾才算獲勝。
- 若扣分後小於 0、等於 1，或未以雙倍收尾使分數為 0，則判定為 **Bust（爆分）**，該回合分數作廢，恢復本回合開始前的總分。

### 9.2 Cricket（板球式飛鏢）

- 僅使用 15–20 與 Bullseye 共 7 個目標區域。
- 每個區域需命中 3 次以「開啟（Open）」，開啟後若對手尚未開啟同區域，額外命中可計分；雙方皆開啟則不再計分（僅計次數）。
- 所有區域開啟且積分最高者獲勝。

### 9.3 Around the Clock（繞圈練習模式，休閒單人）

- 依序命中 1 到 20，再到 Bullseye，完成全部所需鏢數最少者獲勝，適合新手練習瞄準。

### 9.4 玩家設定

- 支援 1–4 名玩家（本機輪流制，暫不含連線對戰）。
- 每位玩家可自訂暱稱、頭像圖示、代表色。

## 10. 音效與音樂規格

### 10.1 BGM 規格（對應需求 6、7）

| 場景 | 曲風 | 音量基準 |
|---|---|---|
| 主畫面 | 鋼琴獨奏，輕快明亮，節奏中等偏快，類似咖啡廳爵士鋼琴風格 | 見下方音量放大規則 |
| 遊戲進行中 | 鋼琴為主旋律，加入輕微弦樂鋪底，節奏輕快不搶戲，避免干擾玩家專注瞄準 | 同上 |
| 結算畫面 | 鋼琴變奏，勝利時明亮上揚、平局時中性溫和 | 同上 |

**音量放大規則（需求 7）**：
遊戲內 BGM 的實際輸出音量須為系統預設基準音量的 **10 倍增益**。實作上因瀏覽器 `<audio>.volume` 上限為 `1.0`，需透過 **Web Audio API 的 `GainNode`** 將增益值設為 `10`（`gainNode.gain.value = 10`），並在 `audioManager.js` 中提供使用者可調整的「主音量」與「BGM 相對倍率」雙層控制，同時加入 **動態壓縮器（DynamicsCompressorNode）** 避免音量放大 10 倍後產生破音（clipping）或爆音，確保放大後音質仍清晰不失真。設定頁的音量滑桿實際控制的是「使用者感知音量」，內部仍以 10 倍增益為基準運算。

### 10.2 音效規格（SFX，對應需求 6）

- 整體音效基調為「**高音、輕脆、清亮**」，類似木琴/玻璃鈴鐺/清脆敲擊聲，避免低沉厚重音效，營造輕鬆愉快的遊戲氛圍。

| 事件 | 音效特性 |
|---|---|
| UI 按鈕點擊 | 短促高音「叮」聲，時長 < 150ms |
| UI 滑過（hover） | 極短「嗒」聲，音量較點擊音低 |
| 飛鏢出手 | 輕微「咻」的空氣音 |
| 命中一般計分區 | 清脆「叩」聲，木質靶面質感 |
| 命中紅心 Bullseye | 高音「叮鈴」聲＋畫面金色閃光特效同步 |
| 命中三倍/雙倍環 | 附加清亮「鏘」音效疊加，強調高分瞬間 |
| Bust 爆分 | 短促下降音階提示（仍維持輕快不刺耳） |
| 遊戲勝利 | 上揚鋼琴琶音＋清脆鈴聲組合 |

- 所有音效需**預先載入（Preload）**於 `audioPreloader.js`，避免遊戲中因即時載入造成延遲卡頓，確保命中瞬間音效零延遲播放。
- 音效需支援**疊音播放**（例如快速連續命中時可同時播放多個音效實例，不互相截斷）。

## 13. 多國語系（i18n）規格（對應需求 9）

### 13.1 支援語言

- 繁體中文（`zh-TW`，預設）
- 英文（`en-US`）
- 日文（`ja-JP`）

### 13.2 實作原則

- 所有畫面文字（不含靶面數字本身）皆抽離至 `assets/data/i18n/*.json`，格式如下：

```json
{
  "menu.start": "開始遊戲",
  "menu.continue": "繼續遊戲",
  "menu.instructions": "說明",
  "menu.settings": "設定",
  "game.yourTurn": "{player} 的回合",
  "game.bust": "爆分！",
  "game.win": "{player} 獲勝！"
}
```

- HTML 元素以 `data-i18n="menu.start"` 標記，由 `i18nApply.js` 於語言切換時統一替換文字內容，支援變數插值（如 `{player}`）。
- 語言切換即時生效，不需重新整理頁面；切換後寫入 `localStorage`，下次開啟自動套用。
- 需特別處理三語言的**字數差異**造成的版面跑版問題（日文/中文較短、英文常較長），按鈕與文字容器統一使用彈性寬度（`min-width` + `padding`），並於 `typography.css` 內針對三語言分別微調 `letter-spacing` 與 `line-height`。

## 14. 存檔與繼續遊戲機制

- 使用 `localStorage` 儲存當前對局狀態：玩家列表、目前分數、回合數、遊戲模式、剩餘鏢數。
- 每次投擲完成後自動存檔（Auto-save），避免瀏覽器意外關閉造成進度遺失。
- 主畫面「繼續遊戲」按鈕讀取此存檔並還原至中斷當下的畫面狀態。
- 若玩家完成整場遊戲（分出勝負），存檔自動清除，「繼續遊戲」按鈕恢復不可點擊狀態。

## 15. 視覺風格與擬真規格（對應需求 13）

- 靶面採用**高解析度木紋外框貼圖 + 麻絮紋理靶面貼圖**，模擬真實酒吧飛鏢盤質感。
- 飛鏢本體具備金屬鏢頭光澤、尼龍鏢尾漸層色彩，命中靶面時呈現**輕微插入深度陰影**與**餘震動畫（晃動 2–3 次後停止）**。
- 背景場景可選擇「酒吧」「電競房」「日式居酒屋」等主題背景，隨配色主題連動切換，強化沉浸感。
- 光影細節：主畫面 Logo 與按鈕具備輕微光澤 hover 效果；靶面命中瞬間有短暫的**聚光燈式高光**掃過效果，強化打擊回饋感。
- 分數彈出動畫：命中後分數以「向上飄出＋淡出」動畫呈現，勝利時觸發彩帶/煙火粒子效果。

## 16. 效能與流暢度規格（對應需求 15）

- 所有動畫優先使用 CSS `transform` / `opacity`，避免觸發 layout reflow，維持 60 FPS。
- 靶面與飛鏢渲染建議採用 **Canvas 2D** 或 **SVG**（依命中判定精度需求擇一，建議 Canvas 以利像素級動畫效果），避免大量 DOM 節點操作造成卡頓。
- 音效與圖片資源皆需**預載（Preloading）**，並提供載入進度畫面（Loading Screen），避免遊戲中途因資源載入造成頓格。
- 行動裝置需針對觸控事件使用 `passive: true` 監聽，並避免不必要的重繪；動畫效果在設定頁提供「簡化特效」開關以支援低階裝置。
- 所有計時器與動畫循環使用 `requestAnimationFrame`，避免使用 `setInterval` 造成掉幀。
- 圖片資源需提供適當壓縮（WebP 優先，PNG 備援），避免資源過大拖慢首次載入時間。

## 17. 無障礙與相容性
