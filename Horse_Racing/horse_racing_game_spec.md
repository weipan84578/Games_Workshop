## 1. 專案概覽

### 1.1 目標

製作一款可在瀏覽器直接執行、無需安裝的賽馬下注模擬遊戲。玩家使用虛擬籌碼對馬匹下注，觀看動態賽馬過程，並透過遊戲內嵌的賽馬知識提升策略判斷能力。

### 1.2 核心特色

- 六匹馬同場競技，動態跑道動畫
- 多維度不確定性系統（天氣、場地、馬況、意外事件）
- Web Audio API 驅動的音效與 BGM
- 完整 RWD，行動裝置觸控友善
- 內嵌真實賽馬知識卡片
- LocalStorage 儲存籌碼與戰績

### 1.3 遊戲流程

```
主畫面 → 選馬與下注 → 比賽直播 → 結果結算 → 回主畫面
             ↑                               ↓
          (可再下注)                    (籌碼更新)
```

## 2. 技術架構

### 2.1 檔案結構

```
horse_racing.html          ← 單一入口，全部程式碼內含
```

### 2.2 模組劃分（皆為 JS 物件/模組，不依賴框架）

| 模組 | 職責 |
|------|------|
| `Router` | 畫面切換，歷史紀錄管理 |
| `GameState` | 全域遊戲狀態（籌碼、戰績、設定） |
| `HorseEngine` | 馬匹資料產生、屬性計算 |
| `RaceSimulator` | 比賽模擬核心，每幀計算位置 |
| `UncertaintySystem` | 天氣、場地、事件隨機產生 |
| `AudioEngine` | Web Audio API，BGM + 音效管理 |
| `Renderer` | Canvas 跑道渲染、動畫迴圈 |
| `BettingSystem` | 賠率計算、下注驗證、結算 |
| `UIHelper` | DOM 操作、Toast 通知、Modal |

### 2.3 依賴

- 無外部 JS 框架
- 無 npm / build step
- Web Audio API（原生）
- Canvas API（原生）
- LocalStorage（原生）
- CSS Custom Properties（原生 RWD）

## 5. 說明畫面 (Guide)

### 5.1 內容章節

**5.1.1 遊戲目標**
> 選擇你認為會贏的馬匹，押注籌碼，觀看比賽，贏取更多籌碼。

**5.1.2 下注方式**

| 注種 | 說明 | 最低賠率 |
|------|------|---------|
| 獨贏 (Win) | 押中第一名 | 1.5x |
| 位置 (Place) | 押中前三名 | 1.2x |
| 連贏 (Quinella) | 押中前二名（不分順序） | 3x |
| 三重彩 (Trifecta) | 押中前三名（需按順序） | 15x |

**5.1.3 馬匹屬性說明**

| 屬性 | 範圍 | 說明 |
|------|------|------|
| 速度 | 1–10 | 基礎奔跑速度 |
| 耐力 | 1–10 | 後段保速能力 |
| 爆發力 | 1–10 | 最後衝刺加速 |
| 穩定性 | 1–10 | 受環境影響的抵抗力 |
| 草地適應 | 1–10 | 草地場地表現加成 |
| 泥地適應 | 1–10 | 泥地場地表現加成 |

**5.1.4 賠率計算**
- 賠率根據馬匹屬性、當日條件、已下注總額動態計算
- 強馬賠率低、弱馬賠率高
- 下注截止前賠率會隨下注分佈浮動

**5.1.5 按鈕說明**

| 按鈕 | 功能 |
|------|------|
| 下一場 | 結算後進入新一場 |
| 重置籌碼 | 籌碼歸零時可重置為 5,000 |
| 知識庫 | 查閱賽馬知識提升勝率 |

### 5.2 版面

- 頂部返回按鈕
- 可折疊的章節（`<details><summary>` 實作）
- 底部「開始遊戲」快捷按鈕

## 6. 知識畫面 (Knowledge)

### 6.1 知識卡片系統

每張知識卡片包含：
- 標題
- 分類標籤（馬匹 / 場地 / 氣象 / 策略 / 歷史）
- 正文說明（200字以內）
- 「對遊戲的影響」欄位（說明如何影響模擬結果）

### 6.2 知識卡片內容（共 15 張）

**[馬匹] 血統與基因**
> 純種馬（Thoroughbred）是現代賽馬最主流的品種，所有現代純種馬皆可追溯至三匹阿拉伯種祖馬：Byerley Turk、Darley Arabian、Godolphin Arabian。血統越純正，速度潛力通常越高，但也可能較為神經質。
> 
> *遊戲影響：高血統馬匹基礎速度加成 +5–15%，但穩定性可能降低。*

**[馬匹] 馬齡與巔峰期**
> 賽馬通常在 3–5 歲達到巔峰，2歲幼馬速度尚未成熟，6歲以上老馬體力開始下滑。不過也有例外，部分老馬憑豐富比賽經驗維持競爭力。
>
> *遊戲影響：年齡屬性影響速度上限與耐力衰退速度。*

**[馬匹] 性格與賽前心情**
> 馬匹情緒對表現影響極大。賽前過度興奮會消耗能量；過度緊張則跑姿失常。頂尖訓練師會在賽前密切觀察馬匹的耳部動作、步態、食慾來判斷當日狀態。
>
> *遊戲影響：每場比賽隨機產生「心情值」(-20% ~ +15%)，影響最終發揮。*

**[場地] 草地 vs 泥地**
> 草地跑道速度較快，對馬蹄衝擊較小；泥地跑道在下雨後變軟，體重較重、步幅較大的馬匹反而有優勢。香港馬場（跑馬地、沙田）以草地為主，部分馬不擅長草地卻在泥地表現出色。
>
> *遊戲影響：場地類型與馬匹適應屬性交叉計算，最高有 +20% 加成或 -15% 懲罰。*

**[場地] 跑道狀況評級**
> 跑道狀況分為：堅硬(Firm) → 良好(Good) → 稍軟(Good to Soft) → 軟地(Soft) → 重地(Heavy)。狀況越軟，需要消耗越多能量，耐力型馬匹更有優勢。
>
> *遊戲影響：跑道狀況影響所有馬匹的耐力消耗速率（最高 ±25%）。*

**[場地] 閘位效應**
> 起跑閘位對短途賽（1000–1200m）影響顯著。內閘（1–3號）可搶佔有利位置，外閘（10號以上）容易被擠到外圈，增加跑動距離。長途賽影響相對較小。
>
> *遊戲影響：閘位隨機分配，內閘馬匹起跑加速 +5–10%，外閘有 -3–8% 懲罰。*

**[氣象] 天氣對賽事的影響**
> 晴天對速度型馬有利；陰天氣溫適中，耐力型馬更舒適；雨天泥濘場地大幅降低整體速度，但利於泥地型馬。強風則影響外欄馬匹，增加額外體力消耗。
>
> *遊戲影響：天氣每場隨機，分晴天/多雲/小雨/大雨四種，各有對應修正係數。*

**[氣象] 溫度與濕度**
> 高溫高濕環境（如夏季香港）對馬匹體力消耗極大，尤其影響長途賽的後段爆發力。部分來自溫帶的馬匹在熱帶氣候下適應期更長。
>
> *遊戲影響：高溫條件下所有馬匹耐力衰減速度加快 10–20%。*

**[策略] 前領策略 vs 後上策略**
> 前領馬（Frontrunner）優先搶佔領跑位置，消耗較多前段體力，容易在最後直路被後上馬追過。後上馬（Closer）保存體力，靠最後300m爆發，但需要同伴製造較快節奏才能奏效。
>
> *遊戲影響：騎師策略隨機分配，影響馬匹各段速度分配比例。*

**[策略] 騎師的重要性**
> 頂尖騎師能在比賽中精準掌握配速、選擇最佳路線、避開擁擠。騎師的即時判斷可彌補馬匹 5–10% 的先天不足，或因失誤讓強馬表現失常。
>
> *遊戲影響：騎師評分隨機，A級騎師加成 +8%，C級騎師有 -5% 懲罰及更高失誤率。*

**[策略] 負磅與讓磅制度**
> 賽事通常規定攜帶重量（負磅），體重較輕的騎師需加磅達到規定。每減少1磅負重，理論上可提升 0.1–0.2 秒成績。讓磅賽讓弱馬攜帶較輕重量以平衡競爭。
>
> *遊戲影響：負磅隨機分配，輕磅馬有小幅速度加成。*

**[歷史] 香港賽馬歷史**
> 香港賽馬始於1846年，由香港賽馬會（HKJC）主辦，是全球最重要的賽馬市場之一。每年冬季賽季吸引全球頂尖馬匹角逐，沙田馬場可容納8萬名觀眾。
>
> *遊戲背景：遊戲場景以香港賽馬為原型設計。*

**[歷史] 世界三大頂級賽事**
> 日本賽馬（JRA）、英國皇家賽馬會（Ascot）、肯塔基德比（Kentucky Derby）並稱世界三大頂級賽事。馬匹在不同國家和賽道的跨國征戰，考驗的不只是速度，更是適應能力。
>
> *遊戲影響：部分馬匹設定為「遠征馬」，有額外適應期懲罰。*

**[馬匹] 訓練與操練成績**
> 正式比賽前的晨操（Morning Trial）成績是判斷馬匹狀態的重要指標。操練時間縮短代表狀態上升；若馬匹最近操練時間明顯變慢，通常預示當日表現不佳。
>
> *遊戲影響：每匹馬顯示「近期操練評分」，影響當場能力的 ±10%。*

**[場地] 距離適性**
> 馬匹有其最適跑距，短途馬（Sprint，1000–1200m）依賴爆發力；中途馬（Mile，1400–1600m）需速度與耐力並重；長途馬（Stayer，2000m以上）以耐力取勝。強迫短途馬跑長途，或讓長途馬跑短途，都會導致表現大幅下滑。
>
> *遊戲影響：每場比賽有固定距離，與馬匹適性不符者有 -15% 懲罰。*

### 8.4 下注限制

- 單注最低：$ 100

- 每場最多：6 注

```
獨贏賠率 = 基礎賠率 × 天氣係數 × 場地係數 × 心情係數 × 閘位係數
```

賠率範圍：獨贏 1.5x–20x，根據馬匹強弱與當日條件動態調整。

## 12. 比賽模擬引擎

### 12.1 物理模型

比賽分三段：

| 段落 | 距離比例 | 主導屬性 |
|------|---------|---------|
| 起跑段 | 0–20% | 速度 + 閘位 + 心情 |
| 中段 | 20–70% | 速度 × 耐力衰減 |
| 衝刺段 | 70–100% | 爆發力 + 剩餘體力 |

### 12.2 速度計算公式

```javascript
// 基礎速度
baseSpeed = (stats.speed × 0.4 + stats.stamina × 0.3 + stats.burst × 0.3) / 10

// 環境修正
envMod = weatherMod × trackMod × distanceMod × gateMod

// 動態修正（每幀）
currentSpeed = baseSpeed × envMod × race.mood × race.fitness
             × jockeyMod × energyMod(horse.energy)
             + randomNoise()   // 每幀加入微小隨機擾動

// 體力衰減
horse.energy -= energyDrain(currentSpeed, trackCondition, horse.stats.stamina)
```

### 12.3 體力衰減模型

```javascript
energyDrain = baseDrain
  × (1 + speedOverBaseline × 0.5)      // 高速超支
  × trackSoftnessPenalty               // 軟地額外消耗
  × (1 - stats.stamina / 20)           // 耐力越高衰減越慢

// 體力影響速度
energyMod = energy > 50
  ? 1.0                                // 體力充足，不影響
  : 0.7 + (energy / 50) × 0.3         // 體力不足，速度下滑
```

### 12.4 最後衝刺

```javascript
// 進入最後 20% 路程
if (progress > 0.80) {
  burstMod = 1 + (stats.burst / 10) × 0.3 × (horse.energy / 100)
  // 體力越充足，爆發越強
}
```

### 14.2 BGM 系統

**BGM 生成策略：程式化合成**

| 場景 | BGM 風格 | 實作方式 |
|------|---------|---------|
| 主畫面 | 輕快爵士 | 低頻 Bass + 高頻和弦循環（OscillatorNode） |
| 下注畫面 | 緊張弦樂 | 多音軌混音，帶節拍器節奏 |
| 比賽中 | 激昂進行曲 | 鼓點 + 旋律交疊，節奏隨比賽進度加快 |
| 最後直路 | 高潮衝刺 | 升調 + 加速，最後10秒 BPM +30% |
| 結果 | 勝利/失敗 | 短暫旋律收尾 |

**BGM 程式化合成範例：**

```javascript
// 鼓點生成（比賽 BGM 核心）
function scheduleDrumBeat(time, bpm) {
  // 大鼓：每拍
  scheduleOsc(60, 'sine', time, 0.1, 0.3)
  // 小鼓：第二、四拍
  scheduleNoise(time + beatLen, 0.05, 0.15)
  // Hi-hat：每半拍
  scheduleNoise(time, 0.02, 0.05, 8000)
}

// 旋律線（下注 BGM）
const melodyNotes = [261, 293, 329, 349, 392, 440, 493, 523]
function scheduleMelody(startTime, bpm) {
  melodyNotes.forEach((freq, i) => {
    scheduleOsc(freq, 'sawtooth', startTime + i * beatLen, 0.1, beatLen * 0.9)
  })
}
```

// 獨贏基礎賠率（根據馬匹綜合評分）
function calcBaseOdds(horse) {
  const score = calcHorseScore(horse)  // 0–100 分
  const maxScore = getMaxScore(allHorses)
  const ratio = score / maxScore

  // 強馬賠率低，弱馬賠率高
  return 1.5 + (1 - ratio) * 15  // 範圍 1.5–16.5

// 動態賠率（受下注分佈影響）
function calcDynamicOdds(horse, totalBets) {
  const horseBets = getBetsOnHorse(horse.id)
  const betRatio = horseBets / totalBets
  return baseOdds × (1 - betRatio × 0.3)  // 熱門馬賠率被壓低
}

| 操練成績 | `race.fitness` 係數 | 操練評級 A 的馬優先考慮 |
