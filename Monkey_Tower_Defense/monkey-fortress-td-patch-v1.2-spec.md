## 1. 數值調整：炸彈猴

> **注意：原規格書將此塔稱為「炸彈猴」，費用原為 80。**  
> 本次改動對應的是「重砲系近距爆炸型」塔，依玩家慣用名稱以「炸彈猴」統一稱呼。

### 1.1 調整內容

| 屬性 | v1.1 | v1.2 | 說明 |
|------|------|------|------|
| 建造費用 | 80 | **90** | 提高建造門檻，避免早期過於普及 |
| 基礎傷害 | 4 | **3** | 降低爆炸初始傷害，強化升級必要性 |
| 射程 | 2.5 | 2.5 | 不變 |
| 射速 | 0.8/s | 0.8/s | 不變 |
| 攻擊類型 | 爆炸 | 爆炸 | 不變 |

### 1.2 設計理由

炸彈猴在前期以低費用 + 高範圍傷害強勢壓制群體波次，削弱基礎傷害可讓玩家必須在中期投入升級資源才能維持效率，同時提高建造費用使玩家在「炸彈猴 vs 飛鏢猴」的選擇上更有取捨感。

### 1.3 調整後升級路線數值（不變）

```
升級 A（爆炸強化）：
  A1 爆炸半徑增大  💰 100  爆炸範圍 +0.5 格
  A2 熱核炸彈      💰 160  傷害 +3（升後傷害 6）
  A3 超級核爆      💰 350  傷害再 +5（升後傷害 11），範圍再 +0.5

升級 B（連鎖強化）：
  B1 連鎖爆炸      💰 120  擊殺後殘餘爆炸（傷害 2，範圍 0.5）
  B2 燃燒殘留      💰 200  爆炸後留下火焰格（每秒傷害 1，持續 3s）
  B3 火焰風暴      💰 420  爆炸同時觸發連鎖燃燒至 3 個相鄰目標

頂點技能「核爆猴」：超大範圍爆炸 + 燃燒殘留疊加，冷卻觸發間隔 8s
```

## 2. 數值還原：火砲猴

### 2.1 說明

v1.1 對火砲猴進行了削弱，本版本 v1.2 **完全還原至 v1.0 原始數值**，不做任何調整。

### 2.2 還原後數值

| 屬性 | v1.1（削弱版）| v1.2（還原版）|
|------|-------------|-------------|
| 建造費用 | 120 | **100** |
| 基礎傷害 | 6 | **8** |
| 射程 | 2.5 | **3** |
| 射速 | 0.4/s | **0.5/s** |
| A1 穿甲效果 | 護甲減傷 30% | **護甲減傷 → 0%** |
| A3 超穿甲 | 護甲減傷 10% | **完全無視護甲** |
| B3 熔岩砲爆炸 | 範圍縮小 20% | **原始範圍** |
| B3 熔岩砲燃燒 | 燃燒傷害 -1 | **原始燃燒傷害** |

### 2.3 頂點技能還原

| 版本 | 頂點技能效果 |
|------|------------|
| v1.1 | 穿甲燃燒砲彈，附加 1.5s 冷卻懲罰 |
| v1.2 | **穿甲燃燒砲彈，無額外限制（完全還原）** |

### 2.4 設計理由

火砲猴的強勢定位在現有的免疫/弱點對照系統下已被自然制衡——鋼殼氣球免疫爆炸、電磁氣球免疫穿甲、絕緣氣球免疫穿甲，使火砲猴不再是無腦萬能塔。因此人為數值削弱已無必要，還原原始數值讓其保持強塔定位但非萬能。

## 3. 數值調整：飛鏢猴

### 3.1 調整內容

| 屬性 | v1.1 | v1.2 | 說明 |
|------|------|------|------|
| 建造費用 | 50 | **35** | 降低門檻，強化入門塔定位 |
| 基礎傷害 | 1 | 1 | 不變 |
| 射程 | 3 | 3 | 不變 |
| 射速 | 1.5/s | 1.5/s | 不變 |

### 3.2 設計理由

飛鏢猴作為入門教學塔，降低費用至 35 讓玩家在初始金幣 150 的條件下，第一波可以放置最多 4 座飛鏢猴而非 3 座，更有餘裕嘗試不同佈陣位置。同時在與炸彈猴（費用 90）的比較下，形成明確的「多點輕攻 vs 少點重攻」戰略選擇。

### 3.3 調整後升級路線（不變）

```
升級 A（穿透強化）：
  A1 雙重飛鏢   💰 120  傷害 +1
  A2 穿透飛鏢   💰 200  子彈可穿透 1 個敵人
  A3 無限穿透   💰 450  子彈無限穿透

升級 B（射速強化）：
  B1 射速提升   💰 100  射速 +0.5/s
  B2 超速射擊   💰 180  射速再 +1.0/s
  B3 三連射     💰 400  每次攻擊射出 3 顆子彈

頂點技能「彈幕猴」：三連射 × 無限穿透，射速再 +50%
```

### 4.2 總覽頁：完整玩法說明

#### 📖 基本目標

```
MonkeyFortress TD 是一款塔防遊戲。
你的目標是在地圖上放置猴子塔，阻止一波又一波的氣球通過路徑到達終點。

每當氣球成功穿越路徑到達終點，你會失去生命值。
生命歸零，遊戲結束。
撐過所有波次且生命值大於 0，即為勝利！
```

#### 💰 金幣系統

```
金幣是放置塔和升級塔的主要資源。

獲得金幣的方式：
  • 擊破氣球：每種氣球擊殺後給予不同數量的金幣
  • 波次完成：每完成一波，獲得固定波次獎勵金幣
  • 提前發動：在上一波尚未清空時手動發動下一波，
              可獲得 +10 提前獎勵金幣
  • 賣出塔：將已放置的塔賣出，取回建造費用的 50%

消耗金幣的方式：
  • 建造塔：從底部塔列點選並放置於地圖空地
  • 升級塔：點擊已放置的塔，選擇升級路線
  • 解鎖頂點技能：A 路線與 B 路線皆升至 3 級後解鎖
```

#### ❤️ 生命值系統

```
遊戲開始時擁有一定數量的初始生命值（依地圖與難度設定而異）。

扣血規則：
  • 氣球成功通過終點，根據氣球種類扣除對應生命
  • 部分強力氣球一次通過可扣除 3~5 點生命
  • BOSS 氣球通過可一次扣除 5 點或更多
  • 生命值歸零時，遊戲立即結束（即使當前波次仍有敵人）

生命值無法在遊戲中自然回復。
史詩長征模式的加成卡可提供少量回血機會。
```

#### 🌊 波次系統

```
敵人以「波次」為單位出現，每波有固定的敵人組合與數量。

波次流程：
  1. 畫面顯示「波次 X / 總波次」與下一波敵人預覽
  2. 倒計時結束後（或玩家手動點擊「發動波次」），敵人開始出現
  3. 波次內所有敵人被消滅或通過終點後，波次結算
  4. 獲得波次完成金幣，進入下一波倒計時
  5. 所有波次清空後，遊戲勝利

手動發動波次：
  • 點擊畫面底部的「▶ 發動波次」按鈕可提前觸發
  • 提前發動可獲得 +10 金幣獎勵
  • 敵人仍然存在時也可以強制發動下一波（雙波共存）
```

#### 🐒 放置塔

```
塔是你防禦氣球的主力武器。

放置步驟：
  1. 從底部塔列點選想要的塔（確認金幣足夠）
  2. 拖曳或點擊地圖上的空白綠色格子
  3. 出現半透明預覽時確認位置，再次點擊確認放置
  4. 若點擊路徑格或其他已放塔的格子，取消放置

放置規則：
  • 只能放置在「空地格」（綠色），不可放置於路徑或障礙格
  • 每張地圖的放置格數量有限，需謹慎規劃
  • 放置後塔立即開始自動攻擊射程內的敵人

取消放置：
  • 選取塔後，點擊路徑以外的任意地方可取消
  • 手機版可點擊右下角的「✕ 取消」按鈕
```

#### ⬆️ 升級系統

```
每座塔都有兩條升級路線（A 路線與 B 路線），各可升級 3 次。

升級步驟：
  1. 點擊已放置的塔，開啟升級面板
  2. 選擇 A 路線或 B 路線的下一級升級
  3. 確認金幣足夠後點擊升級按鈕

升級路線說明：
  • A 路線與 B 路線功能完全不同，針對不同戰術需求
  • 兩條路線完全獨立，可同時升滿（A3 + B3）
  • 例如：飛鏢猴 A 路線強化穿透，B 路線強化射速

頂點技能：
  • A 路線升至 3 級 且 B 路線升至 3 級後，解鎖該塔的頂點技能
  • 頂點技能是 A、B 兩路線效果的強力融合版本
  • 費用較高，但效果顯著提升塔的整體戰力

賣出塔：
  • 可隨時賣出塔，取回建造費用 + 已花費升級費用的 50%
  • 注意：地獄難度下禁止賣出塔
```

#### 🎈 氣球種類

```
氣球有各種不同的特性，了解它們的弱點是取勝關鍵。

基本氣球（依強度排列）：
  🔵 藍色  →  最基本，無特殊能力
  🟢 綠色  →  擊破後釋放一個藍色氣球
  🟡 黃色  →  速度快，需注意
  🔴 紅色  →  血量較多
  ⚫ 黑色  →  免疫爆炸傷害（炸彈猴無效！）
  🔩 鉛色  →  超重護甲 + 免疫冰凍
  💀 BOSS  →  超高血量，多種抗性

特殊氣球（各有剋制關係）：
  • 部分氣球對特定攻擊完全免疫
  • 部分氣球對特定攻擊有額外弱點（傷害加倍）
  • 長按（手機）或滑鼠懸停（桌機）氣球可查看其弱點資訊

建議：波次開始前查看「下一波預覽」圖示，
      提前判斷是否需要調整塔的配置！
```

#### ⚙️ 遊戲速度

```
遊戲支援 5 段速度控制：

  1×  正常速度（預設）
  2×  2 倍速
  3×  3 倍速
  4×  4 倍速
  5×  5 倍速（最高速）

點擊 HUD 頂部的速度按鈕循環切換，或直接點擊對應倍數。
注意：速度越高，敵人移動越快，塔攻擊間隔也相應縮短，
     畫面反應時間更短，操作難度提升。
地獄難度禁用加速功能（固定 1×）。
```

#### 📊 評分系統

```
每局遊戲結束後根據以下指標計算分數：

  擊殺數      × 10 分
  剩餘生命    × 50 分
  剩餘金幣    × 2 分
  完成波次數  × 100 分

最終分數再乘以：
  難度係數（簡單 ×0.7 / 普通 ×1.0 / 困難 ×1.5 / 地獄 ×2.5）
  長度係數（速戰 ×0.8 / 標準 ×1.0 / 史詩 ×1.3）

評分等級：
  ⭐⭐⭐  通關且剩餘生命 ≥ 80%（且未使用加速）
  ⭐⭐    通關且剩餘生命 ≥ 50%
  ⭐      通關（生命 ≥ 1）
  無星    遊戲失敗
```

## 5. 速度控制擴充（最高 5 倍）

### 5.1 速度等級定義

| 按鈕標示 | 速度倍率 | 遊戲迴圈 deltaTime 乘數 |
|---------|---------|----------------------|
| **1×** | 1.0 | × 1.0 |
| **2×** | 2.0 | × 2.0 |
| **3×** | 3.0 | × 3.0 |
| **4×** | 4.0 | × 4.0 |
| **5×** | 5.0 | × 5.0 |

### 5.2 速度影響範圍

速度倍率影響以下所有計時行為：

```javascript
// 遊戲迴圈中統一乘以 gameSpeed 係數
function update(rawDeltaTime) {
  const dt = rawDeltaTime * GameState.speed;

  enemies.forEach(e  => e.move(dt));
  towers.forEach(t   => t.updateCooldown(dt));
  projectiles.forEach(p => p.move(dt));
  effects.forEach(fx => fx.update(dt));
  waveManager.update(dt);
  specialEvents.forEach(ev => ev.update(dt));
}
```

**受影響項目：**
- 敵人移動速度
- 塔攻擊冷卻計時
- 子彈飛行速度
- 特效持續時間（燃燒、冰凍、毒素等）
- 波次間隔倒計時
- 地圖特殊事件計時（沙塵暴、雪崩等）

**不受影響項目：**
- 背景音樂播放速度（音樂維持正常速度）
- UI 動畫（按鈕點擊、面板展開等）
- 暫停面板

### 5.4 速度切換邏輯

```javascript
const SPEED_LEVELS = [1, 2, 3, 4, 5];

// 點擊特定速度按鈕
function setSpeed(level) {
  if (GameState.difficulty === 'hell') return;   // 地獄難度鎖定 1×
  GameState.speed = level;
  updateSpeedButtonUI(level);
}

// 更新按鈕視覺
function updateSpeedButtonUI(active) {
  SPEED_LEVELS.forEach(level => {
    const btn = document.getElementById(`speed-btn-${level}`);
    btn.classList.toggle('active', level === active);
  });
```

## 6. 密技系統：金錢密技

### 6.1 功能說明

在**已進入遊戲畫面**（非主選單、非選圖頁）後，透過鍵盤輸入特定按鍵序列，可以觸發隱藏密技。

**金錢密技：**

```
按鍵序列：← ← → → ←
（方向鍵：左 左 右 右 左）

效果：立即獲得 +999 金幣
```

### 6.2 規則與限制

| 規則 | 說明 |
|------|------|
| 觸發平台 | **僅限 Web 版（桌機瀏覽器）**，手機版與 App 版不支援 |
| 觸發場景 | 必須在**遊戲畫面中**（波次進行中或波次間隔均可）|
| 觸發限制 | **可無限次觸發**，每次 +999 金幣 |
| 金幣上限 | 無上限（可無限疊加）|
| 局間持久 | **金幣僅限當局有效**，遊戲結束（勝利或失敗）後，下一局重新以初始金幣開始 |
| 分數影響 | 使用密技後，**本局分數不計入排行榜**，結算畫面顯示「🔧 密技模式」標記 |
| 存檔影響 | 密技局的星星評分**不覆蓋**正常通關的星星紀錄 |
| 地獄難度 | **可使用**（地獄難度的禁止賣塔、禁加速規則維持不變，但金幣密技可觸發）|

### 6.3 輸入序列偵測實作

```javascript
class CheatCodeDetector {
  constructor() {
    // 金錢密技序列：左 左 右 右 左
    this.MONEY_CHEAT = [
      'ArrowLeft', 'ArrowLeft',
      'ArrowRight', 'ArrowRight',
      'ArrowLeft'
    ];
    this.inputBuffer  = [];
    this.bufferWindow = 2000;   // 2 秒內完成輸入才有效
    this.lastKeyTime  = 0;
    this.isActive     = false;  // 是否在遊戲畫面中
  }

  enable()  { this.isActive = true;  }
  disable() { this.isActive = false; this.inputBuffer = []; }

  onKeyDown(event) {
    if (!this.isActive) return;

    const now = Date.now();

    // 超過 2 秒沒有繼續輸入則重置
    if (now - this.lastKeyTime > this.bufferWindow) {
      this.inputBuffer = [];
    }

    // 只接受方向鍵
    const validKeys = ['ArrowLeft','ArrowRight','ArrowUp','ArrowDown'];
    if (!validKeys.includes(event.key)) return;

    this.lastKeyTime = now;
    this.inputBuffer.push(event.key);

    // 只保留最後 N 個輸入（N = 密技序列長度）
    const len = this.MONEY_CHEAT.length;
    if (this.inputBuffer.length > len) {
      this.inputBuffer = this.inputBuffer.slice(-len);
    }

    // 檢查是否匹配
    if (this.checkMatch(this.MONEY_CHEAT)) {
      this.triggerMoneyCheat();
      this.inputBuffer = [];   // 觸發後清空，可立即再次輸入
    }
  }

  checkMatch(sequence) {
    if (this.inputBuffer.length !== sequence.length) return false;
    return this.inputBuffer.every((key, i) => key === sequence[i]);
  }

  triggerMoneyCheat() {
    GameState.gold        += 999;
    GameState.cheatUsed    = true;   // 標記本局使用過密技
    showCheatNotification();
    playCheatSound();
  }
}

// 初始化
const cheatDetector = new CheatCodeDetector();
document.addEventListener('keydown', e => cheatDetector.onKeyDown(e));

// 進入遊戲畫面時啟用
EventBus.on('GAME_START',  () => cheatDetector.enable());
// 離開遊戲畫面時停用
EventBus.on('GAME_END',    () => cheatDetector.disable());
EventBus.on('RETURN_MENU', () => cheatDetector.disable());
```

```javascript
function playCheatSound(audioCtx, gainNode) {
  const notes = [523, 659, 784, 1047, 1319];   // C5 E5 G5 C6 E6
  notes.forEach((freq, i) => {
    const osc = audioCtx.createOscillator();
    const env = audioCtx.createGain();
    osc.connect(env);
    env.connect(gainNode);
    osc.type = 'square';
    const start = audioCtx.currentTime + i * 0.1;
    osc.frequency.setValueAtTime(freq, start);
    env.gain.setValueAtTime(0.3, start);
    env.gain.exponentialRampToValueAtTime(0.001, start + 0.12);
    osc.start(start);
    osc.stop(start + 0.12);
  });
}
```
