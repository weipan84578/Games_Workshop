## 1. 修正：升級樹系統重設計

### 1.1 問題說明

**原始設計（v1.0）：**
> A 路線最多升 3 級，B 路線最多升 3 級，但「第 3 級互斥」——  
> 任一路線升滿第 3 級後，另一路線最多只能到第 2 級。

**問題根源：**
- 玩家反饋感覺「升滿一條就被懲罰」，體驗不佳
- 第 3 級的互斥邏輯在 UI 上不夠清晰，容易誤導

### 1.2 修正方案：A/B 雙路線各自獨立可升至 3 級

**新規則（v1.1）：**

```
每條路線：Lv 1 → Lv 2 → Lv 3，各自完全獨立
A 路線與 B 路線互不限制，皆可升滿 3 級

最大升級狀態：A3 + B3（雙滿）
最低升級狀態：A0 + B0（未升級）
```

**升級費用重新平衡（因為雙滿可行，費用調高）：**

```javascript
// 各路線費用結構（以飛鏢猴為例）
UpgradeCost = {
  pathA: { lv1: 120, lv2: 200, lv3: 450 },   // 原：100 / 150 / 300
  pathB: { lv1: 100, lv2: 180, lv3: 400 }    // 原：80  / 120 / 250
}
// A3+B3 雙滿總費用約為建造費 × 15 倍，確保不過強
```

### 1.3 新增：第 4 級「頂點技能」（雙路線升滿後解鎖）

升滿 A3 **且** B3 後，解鎖該塔的**唯一頂點技能（Apex Skill）**：

```
A1 → A2 → A3 ─┐
               ├─→ 【頂點技能 S1】（需 A3 + B3）
B1 → B2 → B3 ─┘
```

**頂點技能規則：**
- 費用較高（約為 A3 + B3 費用的 0.8 倍）
- 效果是 A 路線與 B 路線特性的**融合爆發**
- 每座塔只有 1 個頂點技能（不分路線）

### 1.6 升級資料結構更新

```javascript
TowerUpgradeState = {
  pathA: 0 | 1 | 2 | 3,     // A 路線當前等級
  pathB: 0 | 1 | 2 | 3,     // B 路線當前等級
  apexUnlocked: Boolean,     // 頂點技能是否已解鎖（A3+B3）
  apexActivated: Boolean     // 頂點技能是否已購買
}

// 升級驗證函式
function canUpgrade(tower, path, level) {
  if (path === 'A') return tower.upgrade.pathA === level - 1;
  if (path === 'B') return tower.upgrade.pathB === level - 1;
  // 頂點技能
  if (path === 'apex') {
    return tower.upgrade.pathA === 3
        && tower.upgrade.pathB === 3
        && !tower.upgrade.apexActivated;
  }
  return false;
}
```

## 2. 平衡：火砲猴削弱方案

### 2.1 問題說明

**原始數值（v1.0）：**

```
火砲猴：費用 100 / 傷害 8 / 射程 3 / 射速 0.5/s
A3 穿甲：無視所有護甲，對鉛色氣球傷害不減
B3 熔岩砲：爆炸 + 燃燒，接近炸彈猴但單體傷害更高
```

**過強原因：**
- 費用 100 對應傷害 8，DPS 遠超同價位塔
- 穿甲技能讓後期鉛色波完全崩潰
- 射速雖慢，但子彈傷害過高無法補救

### 2.2 削弱項目（v1.1）

| 屬性 | v1.0 | v1.1 | 說明 |
|------|------|------|------|
| 基礎傷害 | 8 | **6** | 降低 25%，DPS 從 4 降至 3 |
| 費用 | 100 | **120** | 提高建造門檻 |
| 射程 | 3 格 | **2.5 格** | 縮小覆蓋範圍 |
| 射速 | 0.5/s | **0.4/s** | 更長間隔，降低突發傷害 |
| A1 穿甲砲彈 | 護甲減傷 → 0% | 護甲減傷 → **30%**（仍有部分減傷）|
| A3 超穿甲 | 完全無視護甲 | **護甲減傷 → 10%**（近乎無視但非全免）|
| B3 熔岩砲 | 爆炸 + 燃燒 | 爆炸半徑 **縮小 20%**，燃燒傷害 -1 |

### 2.3 頂點技能同步調整

| 版本 | 頂點技能效果 |
|------|------------|
| v1.0（原設計）| 穿甲燃燒砲彈（過強）|
| v1.1（修正後）| 穿甲燃燒砲彈，但**每次射擊有 1.5 秒額外冷卻**（強調計謀性而非火力輾壓）|

### 2.4 補償：火砲猴定位重新明確

削弱後，火砲猴定位從「全能輸出」調整為「**對裝甲單體精英殺手**」：

- 對**護甲敵人（鉛色、BOSS）** 仍是最佳選擇
- 對**無護甲群體**建議改用炸彈猴或藤蔓猴
- 適合少量精準放置，不適合大量堆疊

                         // |'thunder'|'sonic'|'flame'|'oracle'|'vortex'

### 8.4 傷害計算函式更新

```javascript
function calcDamage(projectile, enemy) {
  // 免疫檢查
  if (enemy.immunities.includes(projectile.attackType)) return 0;

  let dmg = projectile.damage;

  // 護甲減傷
  if (enemy.armor) {
    const pierceLevel = projectile.pierceLevel || 0;   // 0=無穿甲, 1=輕穿甲, 2=重穿甲
    const armorReduction = Math.max(0, enemy.armorValue - pierceLevel * 0.3);
    dmg *= (1 - armorReduction);
  }

  // 弱點加成
  if (enemy.weaknesses[projectile.attackType]) {
    dmg *= enemy.weaknesses[projectile.attackType];
  }

  // 磁化標記加成
  if (enemy.magnetized) dmg *= 1.5;

  // 詛咒標記加成
  if (enemy.cursedByOracle) dmg *= 1.25;

  return Math.ceil(dmg);
}
```
