## 1. 專案概述

### 1.1 專案目標

開發一款純前端、零後端依賴的 **21點（Blackjack）** 撲克牌遊戲，支援：

- **1 名人類玩家** vs **1 ～ 3 名 AI 玩家**
- 行動裝置友善（手機可單手操作）
- 真實撲克牌視覺風格（含花色符號）
- 完整音樂 + 音效系統
- 可自訂難度、音量、玩家人數的設定頁面

### 1.2 核心原則

| 原則 | 說明 |
|------|------|
| 零依賴 | 不引入任何 npm 套件或 CDN 框架，僅使用原生 Web API |
| 可讀字體 | 主畫面、設定頁面字體最小 `18px`，重要資訊 `24px` 以上 |
| 離線可用 | 所有音效以 Web Audio API 動態合成，不依賴外部音頻檔 |
| 真實牌面 | 每張牌以 SVG 或 HTML/CSS 繪製，含點數、花色、角落索引 |

## 4. 遊戲規則設計

### 4.1 標準規則

| 項目 | 規格 |
|------|------|
| 牌組數量 | 預設 **6 副**，可於設定調整（1 / 2 / 4 / 6 副） |
| Ace 點數 | 1 或 11，自動取最大不爆牌值 |
| 人頭牌（J/Q/K） | 各 10 點 |
| Blackjack 賠率 | 3:2（若莊家同時 Blackjack 則平手） |
| 莊家規則 | 軟 17 停牌（Soft 17 stand） |
| Double Down | 初始 2 張牌可加倍 |
| Split | 相同點數可分牌（最多分 3 次） |
| Insurance | 莊家明牌為 A 時可購買保險（1:2） |
| Surrender | 不支援（簡化規則） |

### 4.2 籌碼系統

```
初始籌碼：$1,000
最小下注：$10
最大下注：$500（或當前籌碼，取小）

籌碼面額：$10 / $25 / $50 / $100 / $500
```

### 4.3 遊戲流程狀態

```
IDLE → BETTING → DEALING → PLAYER_TURN → AI_TURN → DEALER_TURN → SETTLEMENT → IDLE
```

各狀態觸發條件與合法操作詳見 [§10 狀態管理](#10-狀態管理)。

## 5. AI 演算法規格

### 5.1 難度等級

| 難度 | 演算法策略 |
|------|-----------|
| 簡單（Easy） | 固定停在 15 點以上 |
| 普通（Normal） | 基礎 Basic Strategy 表 |
| 困難（Hard） | 完整 Basic Strategy + 牌組計算（Hi-Lo Card Counting） |

### 5.2 Basic Strategy 決策表（Normal / Hard）

AI 根據「手牌點數」× 「莊家明牌」查表決定動作：

#### 硬牌（Hard Hand）決策

| 手牌點數 | 莊家 2 | 莊家 3 | 莊家 4 | 莊家 5 | 莊家 6 | 莊家 7 | 莊家 8 | 莊家 9 | 莊家 10 | 莊家 A |
|---------|--------|--------|--------|--------|--------|--------|--------|--------|---------|--------|
| 8 以下   | H | H | H | H | H | H | H | H | H | H |
| 9        | H | D | D | D | D | H | H | H | H | H |
| 10       | D | D | D | D | D | D | D | D | H | H |
| 11       | D | D | D | D | D | D | D | D | D | H |
| 12       | H | H | S | S | S | H | H | H | H | H |
| 13-14    | S | S | S | S | S | H | H | H | H | H |
| 15       | S | S | S | S | S | H | H | H | H | H |
| 16       | S | S | S | S | S | H | H | H | H | H |
| 17+      | S | S | S | S | S | S | S | S | S | S |

> `H` = Hit（要牌）、`S` = Stand（停牌）、`D` = Double（若不可加倍則要牌）

#### 軟牌（Soft Hand，含 A）決策

| 手牌 | 莊家 2-6 | 莊家 7-8 | 莊家 9-A |
|------|---------|---------|---------|
| A+2 / A+3 | H | H | H |
| A+4 / A+5 | D(H) | H | H |
| A+6       | D(H) | H | H |
| A+7       | S / D(S) | S | H |
| A+8+      | S | S | S |

### 5.3 困難模式：Hi-Lo 計牌系統

```javascript
// 牌值計分規則
卡牌 2-6  → +1（有利玩家）
卡牌 7-9  → 0（中性）
卡牌 10-A → -1（有利莊家）

真實計數（True Count） = 運行計數 / 剩餘牌組數

決策調整（True Count ≥ +2）：
  - 更積極 Double Down
  - 在莊家明牌 2-3 時對 16 點停牌（正常應 Hit）
  - Insurance 當 True Count ≥ +3 時購買
```

### 5.4 AI 思考延遲

```
簡單：800ms 固定延遲
普通：600ms ± 200ms 隨機延遲
困難：300ms ± 100ms 隨機延遲（模擬快速計算）
```

### 5.5 AI 玩家個性設定

每位 AI 有獨立名稱與視覺風格，最多同時存在 3 位：

| 識別碼 | 預設名稱 | 頭像風格 | 性格說明 |
|--------|---------|---------|---------|
| AI_1 | Victor | 🎩 紳士 | 謹慎、慣用 Basic Strategy |
| AI_2 | Luna | 🌙 神秘 | 中等冒險，偶爾 Double Down |
| AI_3 | Blaze | 🔥 豪放 | 激進下注，常 Hit 到高點 |

```
│ ♠             │
│               │
│       ♠       │  ← 中央大花色符號（字體 48-64px）
```

| 花色 | 符號 | 顏色 | Hex |
|------|------|------|-----|
| 黑桃（Spades） | ♠ | 深黑 | `#1a1a1a` |
| 紅心（Hearts） | ♥ | 深紅 | `#CC0000` |
| 方塊（Diamonds）| ♦ | 深紅 | `#CC0000` |
| 梅花（Clubs） | ♣ | 深黑 | `#1a1a1a` |

## 8. 音樂與音效規格

> 所有音頻以 **Web Audio API** 動態合成，無需外部音頻檔案。

### 8.1 背景音樂

使用 Web Audio API 的 `OscillatorNode` + `GainNode` 合成爵士風格背景音樂：

```javascript
// 音樂參數
BPM：        90
調式：        C大調
樂器合成：
  - Bass（低音）：方波 OscillatorNode，frequency 65-130Hz
  - Chords（和弦）：三角波，頻率疊加
  - Melody（旋律）：正弦波，隨機行走旋律線
  
和弦進行：    Cmaj7 → Am7 → Dm7 → G7（循環）
循環長度：    16 bars（約 43 秒）
```

**音樂狀態切換：**

| 遊戲狀態 | 音樂變化 |
|---------|---------|
| 主畫面 | 完整爵士旋律，音量 100% |
| 遊戲中 | 節奏更輕，音量降至 60% |
| 結算贏 | 短促勝利旋律（2秒）後恢復 |
| 結算輸 | 降調短旋律（1.5秒）後恢復 |

### 8.2 音效清單

| 事件 | 音效描述 | 合成方式 | 時長 |
|------|---------|---------|------|
| 發牌 | 紙牌滑動聲 | 白噪音 + 快速衰減 | 200ms |
| 翻牌 | 紙牌翻轉拍打 | 粉紅噪音 + 短衝擊 | 150ms |
| 要牌 | 輕拍桌聲 | 低頻衝擊 + 衰減 | 100ms |
| 下注（放籌碼）| 硬幣碰擊 | 高頻金屬音 + 泛音 | 250ms |
| 清除下注 | 輕柔滑動 | 低頻掃描聲 | 200ms |
| 玩家獲勝 | 歡呼短音效 | C大調分解和弦上行 | 800ms |
| 玩家輸牌 | 失落短音效 | A小調下行音階 | 600ms |
| Blackjack | 特殊彩蛋音效 | 華麗音階 + 鈴聲 | 1500ms |
| 按鈕點擊 | 輕柔確認聲 | 正弦波短音（800Hz） | 80ms |
| 籌碼不足 | 警告聲 | 方波低沉（200Hz）×2 | 300ms |
| 加倍/分牌 | 中等力度音效 | 和弦 + 衰減 | 400ms |

### 8.3 音量控制架構

```
MasterGainNode（0.0 ~ 1.0）
├── MusicGainNode（0.0 ~ 1.0）← 設定：音樂音量
└── SFXGainNode（0.0 ~ 1.0） ← 設定：音效音量
```

### 8.4 使用者互動觸發

iOS / Android 限制 Web Audio 需要使用者手勢才能啟動，規格：

- 第一次點擊任意按鈕時，呼叫 `audioContext.resume()`
- 顯示小提示：「🎵 點擊任意位置啟動音效」
- 啟動後隱藏提示

## 10. 狀態管理

### 10.1 全域 GameState 物件

```javascript
const GameState = {
  phase: 'IDLE',          // 當前遊戲階段
  deck: [],               // 剩餘牌組
  players: [],            // 所有玩家（含AI）
  dealer: null,           // 莊家物件
  currentPlayerIndex: 0,  // 當前行動玩家
  round: 0,               // 回合數
  settings: {             // 從 localStorage 讀取
    aiCount: 1,
    difficulty: 'normal',
    deckCount: 6,
    startingChips: 1000,
    musicVolume: 0.8,
    sfxVolume: 0.6
  }
}
```

### 10.2 遊戲階段與合法操作

| 階段 | 說明 | 玩家可操作 |
|------|------|-----------|
| `IDLE` | 等待下注 | 放置籌碼、清除、全押 |
| `BETTING` | 下注確認 | 確認下注（開始發牌） |
| `DEALING` | 發牌動畫 | 無（等待動畫完成） |
| `PLAYER_TURN` | 玩家回合 | 要牌、停牌、加倍、分牌 |
| `AI_TURN` | AI 回合 | 無（等待 AI 決策） |
| `DEALER_TURN` | 莊家翻牌 | 無（自動執行） |
| `SETTLEMENT` | 結算 | 繼續、返回主選單 |

### 10.3 玩家物件結構

```javascript
const Player = {
  id: 'player' | 'ai_1' | 'ai_2' | 'ai_3',
  name: String,
  isHuman: Boolean,
  chips: Number,
  bet: Number,
  hands: [Hand],          // 分牌時可有多手
  activeHandIndex: 0,
  status: 'waiting' | 'playing' | 'bust' | 'stand' | 'blackjack'
}
```

### 10.4 Hand 物件結構

```javascript
const Hand = {
  cards: [Card],
  value: Number,          // 自動計算最佳點數
  isSoft: Boolean,        // 是否含彈性 Ace
  isBlackjack: Boolean,
  isBust: Boolean,
  bet: Number             // 分牌時各手獨立下注
}
```

### 11.1 發牌動畫

```css
@keyframes dealCard {
  from {
    transform: translate(var(--deck-x), var(--deck-y)) scale(0.5) rotate(-15deg);
    opacity: 0;
  }
  to {
    transform: translate(0, 0) scale(1) rotate(0deg);
    opacity: 1;
  }

/* 時長：300ms，easing：cubic-bezier(0.34, 1.56, 0.64, 1) */
```

### 11.2 翻牌動畫（3D）

```css
/* 牌背翻轉為牌面 */
@keyframes flipCard {
  0%   { transform: rotateY(0deg); }
  50%  { transform: rotateY(90deg); }
  100% { transform: rotateY(0deg); }
}
/* 在 50% 時切換為牌面內容 */
/* 時長：500ms */
```

### 11.3 籌碼堆疊動畫

- 籌碼放置時有彈跳效果（`cubic-bezier` 彈性曲線）
- 籌碼依金額大小呈現不同顏色（$10=白、$25=綠、$50=藍、$100=黑、$500=紫）

### 11.4 結算特效

| 結果 | 特效 |
|------|------|
| 玩家獲勝 | 金幣雨動畫（CSS particles）+ 閃光 |
| Blackjack | 全螢幕金光 + 粒子爆炸 |
| 玩家輸牌 | 螢幕輕微震動（`translateX` 抖動） |
| 平手（Push）| 籌碼滑回動畫 |

### 11.5 點數更新動畫

- 新增一張牌後，點數以 `countUp` 方式滾動更新
- 超過 21 點時，點數變紅色 + 輕微放大脈衝
