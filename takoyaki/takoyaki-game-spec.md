## 1. 專案總覽

### 1.1 遊戲簡介
玩家扮演章魚燒攤販老闆，體驗從「倒麵糊 → 放章魚 → 加配料 → 翻轉 → 出爐 → 淋醬裝飾 → 出餐」的完整章魚燒製作流程。畫面走「可愛日系攤販」風格，操作簡單直覺，適合大人小孩、手機與電腦都能輕鬆遊玩。

### 1.2 核心技術原則
| 項目 | 規格 |
|---|---|
| 執行方式 | **純前端**，雙擊 `index.html` 即可開啟，**不需要 build、不需要 npm install、不需要啟動任何 server** |
| 技術棧 | HTML5 + CSS3 + Vanilla JavaScript（ES6+），**不使用**任何需要編譯的框架（不用 React/Vue 的 build 流程、不用 Webpack/Vite） |
| 外部資源 | 若使用 CDN（如字型），需同時提供本地備援字型，避免離線時破版 |
| 瀏覽器支援 | Chrome / Safari / Edge / Firefox 最新兩個版本，iOS Safari、Android Chrome |
| 儲存方式 | `localStorage`，用於「繼續遊戲」進度、設定、語言、音量、解鎖成就 |

### 2.1 index.html 引入原則
- `<head>` 內僅引入：`css/main.css`、必要的本地字型 `@font-face`（寫在 CSS 內，不寫在 HTML）。
- `<body>` 結尾僅引入：`js/app.js`（設定 `type="module"`，由它以 `import` 方式載入其餘模組）。
- **index.html 本身不得包含任何遊戲邏輯 inline script／inline style**，只保留畫面容器（如 `<div id="screen-main-menu">`、`<div id="screen-game">` 等空殼容器），內容由 JS 動態渲染或以 `<template>` 標籤預放。

## 3. RWD（響應式設計）規格

### 3.1 斷點規劃
| 裝置類型 | 寬度範圍 | 版面策略 |
|---|---|---|
| 手機直式 | ≤ 480px | 單欄、烤盤置中放大、操作鈕移至畫面下方安全區 |
| 手機橫式 / 小平板 | 481–768px | 烤盤置中、左右保留裝飾留白 |
| 平板 | 769–1024px | 維持手機版比例但元件放大 20% |
| 桌機 | ≥ 1025px | 左右加寬留白＋裝飾插畫，遊戲核心區域維持固定比例（不無限拉伸變形） |

### 3.2 核心原則
1. 遊戲主要「烤盤區域」使用 `aspect-ratio` 鎖定比例，避免拉伸變形。
2. 所有互動元件（按鈕、翻轉工具）最小點擊區域 **44×44px**（符合行動裝置觸控標準）。
3. 使用 `dvh` / `svh` 取代 `vh`，避免手機瀏覽器網址列造成版面跳動。
4. 行動裝置操作按鍵（翻面、加料、出爐）固定於畫面「下方安全區」，並：
   - 使用**半透明浮動列**，不覆蓋烤盤主視覺區。
   - 提供「收合／展開」小箭頭，玩家可暫時收起操作列查看完整畫面。
   - 使用 `env(safe-area-inset-bottom)` 避開 iPhone Home Indicator。
5. 橫向/直向切換時（`orientationchange`）即時重新計算版面，不需重整頁面。
6. 文字與圖示等比例縮放（使用 `clamp()` 函式設定字級與間距）。

## 4. 視覺與 UI 規範

### 4.1 字體規格
- 全站字體一律偏大、清晰易讀：
  - 內文最小 `18px`（手機）／`20px`（桌機）。
  - 標題 `28px–40px`（依 `clamp()` 依裝置縮放）。
  - 按鈕文字 `20px` 以上，字重 `600–700`。
- 多語系分流字型：
  - 中文：圓體感黑體（如 Noto Sans TC / 圓體）。
  - 日文：Noto Sans JP。
  - 英文：Fredoka / Baloo 2（圓潤可愛感）。
- 所有文字需有輪廓或陰影輔助（`text-shadow`），確保在彩色背景上清晰可辨，杜絕「看不到字」的情況。

### 4.2 配色主題（可切換，設定頁選擇）
| 主題名稱 | 主色 | 輔色 | 強調色 | 風格說明 |
|---|---|---|---|---|
| 🌸 可愛粉（預設） | #FFD6E8 | #FFF0F5 | #FF6FA5 | 大量愛心、星星、章魚燒吉祥物貼紙 |
| 🌊 海洋藍 | #CDEFFF | #F0FBFF | #2196C9 | 章魚主題延伸，泡泡、貝殼裝飾 |
| ☀️ 元氣黃 | #FFF3C4 | #FFFDF0 | #FFB300 | 祭典燈籠、太陽花裝飾 |
| 🍵 抹茶綠 | #DFF3E3 | #F3FBF4 | #4CAF6D | 和風竹葉、豆沙裝飾 |
| 🌙 夜間紫（護眼模式） | #2B2140 | #3A2E55 | #B79CFF | 星空、燈籠夜市風格 |

- 每個主題皆定義：背景色、卡片背景、主要按鈕色、次要按鈕色、成功色、警示（燒焦）色、文字色、陰影色，並全部落在 CSS 變數（`--color-*`），主題切換只需替換 CSS class（如 `<body class="theme-cute-pink">`）。
- 所有配色皆需通過「對比度檢查」（WCAG AA 以上），避免淡色文字疊在淡色背景上看不清楚。

### 4.3 可愛風主題元素（大量可愛圖案）
- 章魚燒角色化：Q 版章魚吉祥物（開心、驚訝、燒焦哭臉等表情差分）。
- 裝飾圖示：愛心、星星、雲朵、小魚旗、燈籠、蒸氣泡泡、閃亮 sparkle 特效。
- 按鈕造型：圓角＋Q彈按壓動畫（按下時 `scale(0.95)` + 彈跳回彈）。
- 載入畫面：章魚燒角色轉圈圈的可愛 Loading 動畫。

### 4.4 擬真與豐富度
- 烤盤：以分層貼圖＋CSS 動畫模擬「麵糊倒入→冒泡→半凝固→翻轉→金黃酥脆→燒焦」的漸層色澤變化（非單張圖片替換，而是漸層濾鏡＋貼圖疊加）。
- 加入油煙效果（半透明白色粒子飄動動畫）、翻轉章魚燒時的拋物線動畫與旋轉。
- 出爐時加上柴魚片「隨熱氣飄動」的細節動畫（CSS `@keyframes` 輕微擺動）。
- 背景加入日式攤販街景（燈籠搖曳、招牌旗幟飄動、路人剪影走動的視差捲動 Parallax，桌機版可見，手機版簡化以維持效能）。

## 5. 主畫面（Main Menu）規格

### 5.1 功能項目（僅 4 個，保持簡潔）
1. ▶️ **開始遊戲**（New Game）— 若已有存檔，需二次確認是否覆蓋。
2. ⏯️ **繼續遊戲**（Continue）— 無存檔時此按鈕顯示為灰階不可點擊狀態，並提示「尚無進度」。
3. 📖 **遊戲說明**（How To Play）
4. ⚙️ **設定**（Settings）

### 5.2 主畫面設計原則
- **不得放置任何可調整設定的選項在主畫面**（音量、語言等一律收進「設定」頁面，符合需求 14）。
- 主畫面僅保留：遊戲 Logo（可愛章魚燒吉祥物 + 標題字樣）、上述 4 個按鈕、背景裝飾動畫。
- 語言切換入口統一放在「設定」頁，但主畫面首次載入時依瀏覽器 `navigator.language` 自動偵測預設語言。
- 按鈕排列採垂直置中，手機/桌機皆保持置中不跑版；背景可有輕微動態（雲朵飄動、燈籠搖晃），但不可干擾按鈕點擊。

## 6. 遊戲說明（How To Play）頁面規格

### 6.1 結構
- 採**分頁式／步驟式卡片**設計，而非長篇文字堆疊，確保「整潔、大量圖示、易讀」。
- 建議分為以下段落，每段落搭配對應可愛圖示／小動畫示意圖：

| 步驟 | 圖示 | 說明重點 |
|---|---|---|
| 1️⃣ 倒麵糊 | 🥄＋麵糊桶圖示 | 點擊/拖曳倒入麵糊到烤孔 |
| 2️⃣ 放章魚 | 🐙 圖示 | 將章魚塊放入每個烤孔中央 |
| 3️⃣ 加配料 | 🧅🥓 圖示 | 蔥花、紅薑、天婦花選擇加入 |
| 4️⃣ 翻轉時機 | ⏱️＋章魚燒針圖示 | 顏色變化提示翻面時機，太晚會燒焦 |
| 5️⃣ 出爐裝盤 | 🍽️ 圖示 | 完成後裝盤 |
| 6️⃣ 淋醬裝飾 | 🍯🧂 圖示 | 醬汁、美乃滋、柴魚片、海苔粉 |
| 7️⃣ 評分與星星 | ⭐⭐⭐ 圖示 | 依熟度、配料完整度、時間評分 |
| 操作說明（RWD） | 📱🖱️ 圖示 | 分別列出「手機觸控操作」與「電腦滑鼠操作」對照圖 |

### 6.2 呈現方式
- 每個步驟為一張圓角卡片，圖示置左（或置上，依 RWD 切換）、文字置右，字體加大、行距寬鬆。
- 提供左右箭頭或滑動（swipe）切換步驟，並有進度圓點指示（如 ● ● ○ ○）。
- 底部提供「返回主畫面」按鈕，圖示明確（← 圖示 + 文字）。
- 所有說明文字需支援三語系切換（與設定頁語言選項連動）。

## 7. 設定（Settings）頁面規格

### 7.1 排版原則
- 採**單欄清單式**排版，乾淨簡單，每個設定項目一列，避免視覺雜亂。
- 分組區塊：
  1. 🔊 **音效設定**：BGM 音量／音效音量（各自獨立滑桿，圓形滑塊、可愛配色、即時試聽音效）。
  2. 🎨 **主題配色**：橫向可滑動的色卡選擇列（5 個主題色點，選中時有可愛外框動畫）。
  3. 🌐 **語言設定**：日文／英文／中文三選一，搭配國旗小圖示 + 語言名稱按鈕。
  4. 💾 **資料管理**：清除進度／重置設定（需二次確認彈窗，避免誤觸）。
- 每個滑桿、按鈕都需有清楚的可視回饋（拖曳時數值即時顯示、點擊時有 icon 高亮動畫）。
- 設定頁需有明顯「返回」按鈕（左上角或底部），所有變更**即時生效並自動儲存**至 `localStorage`（不需另外按「儲存」）。

### 7.2 元件視覺規格
- 滑桿：軌道使用主題色系淺色，滑塊為圓形可愛圖示（如章魚燒造型的小滑塊）。
- 選中狀態一律使用「勾勾 icon ✔️」或「外框發光」表示，不可只靠顏色些微差異表示（避免色弱使用者辨識困難）。

## 8. 音樂與音效規格（BGM / SFX）

### 8.1 BGM 規格
- 全部採用**輕快鋼琴曲風**，共規劃 4 首：
  1. 主選單用（明亮、歡迎感）
  2. 遊戲進行用 A（俏皮、有節奏感，適合手忙腳亂做章魚燒）
  3. 遊戲進行用 B（溫馨、悠閒風格，交替播放避免單調）
  4. 慶祝/過關用（短促歡快，過關時插播後再回到原 BGM）
- BGM 採**清單隨機＋不重複播放**邏輯（`playlist.js` 控制），避免玩家聽膩同一首。
- 音樂需可無縫循環（loop 開頭結尾銜接處理過，避免破音卡頓）。

### 8.2 音效規格
- 全部使用**高音、清脆、輕盈**的音效素材（如鈴鐺聲、木琴聲、清脆水滴聲），避免低沉、刺耳或過於厚重的音效，維持整體「可愛」調性。
- 音效清單：倒麵糊聲、滋滋烤聲（可循環，低音量背景音效）、翻轉「叮」聲、裝盤聲、淋醬「唰」聲、按鈕點擊「啵」聲、成功「鈴鐺」聲、燒焦提示「短音警示」（需明確但不刺耳）、得星星「亮晶晶」音效。

### 8.3 音量規格（**重點需求**）
> **需求 7：遊戲中的 BGM 音量都放大為原來的 10 倍。**

- 實作方式：`audio-manager.js` 內建「BGM 增益係數 `BGM_GAIN_MULTIPLIER = 10`」。
- 由於瀏覽器 `<audio>` 標籤 `volume` 屬性上限為 `1.0`，無法單純乘以 10 直接超過原生上限，因此採用 **Web Audio API 的 `GainNode`** 實作真正的音量放大：
  ```js
  const audioCtx = new (window.AudioContext || window.webkitAudioContext)();
  const source = audioCtx.createMediaElementSource(bgmAudioElement);
  const gainNode = audioCtx.createGain();
  gainNode.gain.value = userVolumeSetting * 10; // 10 倍增益
  source.connect(gainNode).connect(audioCtx.destination);
  ```
- 設定頁的 BGM 滑桿數值（0–100%）仍對應「使用者感受音量」，實際輸出時內部再乘以 10 倍增益係數，讓遊戲中的 BGM 顯著比一般網頁音樂更響亮飽滿。
- 需加入**音量安全提示**：首次進入遊戲若音量過大，跳出可愛小提醒「音樂音量較大唷，建議先調整裝置音量再開始遊戲！」，避免使用者被突然的大音量嚇到。
- SFX（音效）維持一般音量邏輯（0–100% 對應正常 `volume`），僅 BGM 套用 10 倍增益，兩者音量各自獨立滑桿控制。

## 9. 多國語系規格（i18n）

### 9.1 支援語言
- 🇯🇵 日文（ja）
- 🇺🇸 英文（en）
- 🇹🇼 繁體中文（zh-TW）

### 9.2 實作方式
- 所有畫面文字**不得寫死於 HTML／JS 邏輯中**，一律透過 `key` 對應至 `lang-zh.js` / `lang-en.js` / `lang-jp.js` 字典檔取得對應字串。
- 範例字典結構：
  ```js
  export default {
    menu_start: "開始遊戲",
    menu_continue: "繼續遊戲",
    menu_howto: "遊戲說明",
    menu_settings: "設定",
    game_flip_hint: "該翻面囉！",
    settings_bgm_volume: "背景音樂音量",
    // ...
  };
  ```
- `i18n-engine.js` 提供 `t(key)` 函式供各模組呼叫，並在語言切換時觸發全畫面文字即時更新（不需重新整理頁面）。
- 語言切換入口置於設定頁，選擇後立即儲存於 `localStorage`，下次開啟遊戲自動套用上次選擇的語言。
- 日文／中文／英文的**字級與行距個別微調**（日文假名偏小需適度放大、中文筆畫多需加寬字距），確保三語系下畫面都不跑版、不擁擠。

## 10. 遊戲流程與玩法規格

### 10.1 基本遊戲循環
1. 選擇關卡／自由模式。
2. 烤盤上有 6–12 個烤孔（依關卡難度）。
3. 依序：倒麵糊 → 放章魚塊 → （可加蔥花/紅薑等配料）→ 等待半熟提示 → 翻面 → 等待全熟提示 → 出爐裝盤 → 選擇醬料組合裝飾。
4. 系統依「熟度精準度」「配料完整度」「操作時間」給予 1–3 星評分，並有可愛的星星飛入動畫與音效。
5. 完成關卡後可查看成果小圖鑑／進入下一關。

### 10.2 狀態機（單一烤孔）
```
生麵糊(raw) → 半熟(half-cooked) → [提示翻面] → 已翻面(flipped)
   → 熟成(cooked) → [提示出爐] → 完成(done)
半熟/已翻面若超時未處理 → 燒焦(burnt)（可愛哭臉表情＋短促警示音）
```

### 10.3 進度儲存（繼續遊戲）
- 每完成一個關卡或每隔固定操作次數，自動將目前關卡進度、分數、星星數存入 `localStorage`。
- 主畫面「繼續遊戲」讀取該存檔，若無存檔則按鈕呈現灰階禁用狀態並提示文字（多語系）。

## 11. 效能與流暢度規格

- 所有動畫優先使用 CSS `transform` / `opacity`，避免觸發 layout reflow，確保 60fps 流暢度。
- 遊戲主迴圈使用 `requestAnimationFrame`，避免使用 `setInterval` 造成掉幀。
- 圖片素材使用 SVG（向量圖示為主）或壓縮過的 WebP／PNG，控制單一素材檔案大小（建議 < 100KB／張）。
- 音效／音樂採預先載入（preload）機制，避免遊戲進行中因載入音檔造成卡頓。
- 行動裝置版本降低背景視差層數與粒子特效數量，確保中低階手機也能流暢運行。
- 章魚燒烤孔數量較多時（12 孔），使用物件池（Object Pool）重複利用 DOM 節點或 Canvas 繪製，避免頻繁 DOM 新增/刪除造成的效能問題。
