# Doodle Jump（塗鴉跳躍）純前端遊戲完整規格書

> 文件類型：產品需求規格（PRD）＋技術設計規格（TDS）＋測試與驗收規格  
> 文件版本：1.0.0  
> 目標平台：桌面瀏覽器、平板、手機瀏覽器  
> 執行方式：直接雙擊專案根目錄的 `index.html`，以 `file://` 開啟即可遊玩  
> 專案限制：不需安裝套件、不需 build、不需啟動任何 server、不建立 `README.md`

## 1. 專案摘要

本專案是一款受經典垂直跳躍玩法啟發的單人休閒網頁遊戲。玩家控制可愛角色持續向上跳躍，踩踏不同平台、收集道具、避開障礙並挑戰最高分。整體視覺採「可愛角色＋帶有現實光影與景深的天空世界」混合風格，兼顧清楚易讀、豐富、有趣與長時間遊玩舒適度。

遊戲必須完全離線可用。所有 HTML、CSS、JavaScript、字型、圖片與音訊均存放於專案內，不依賴 CDN、API、遠端字型、登入服務或網路連線。

### 1.1 核心價值

- 開箱即玩：雙擊 `index.html` 即可開始。
- 跨裝置順暢：桌機、平板與手機皆有不遮擋遊戲畫面的操作方式。
- 介面清楚：大字體、高對比、按鈕狀態明確。
- 可愛且豐富：大量原創可愛圖示、角色表情、粒子與場景細節。
- 有持續挑戰性：難度隨高度提升，具本機存檔與本機排行榜。
- 可驗證：附可直接開啟的瀏覽器單元測試頁面，測試核心規則與各模組。

### 1.2 非目標範圍

- 不實作多人連線、雲端同步、帳號登入或全球線上排行榜。
- 不加入廣告、付費、追蹤碼或分析服務。
- 不要求 PWA 安裝、Service Worker 或離線快取；因為檔案本身即為本機資源。
- 不複製任何受版權保護的原作圖片、音樂、字型或商標素材；玩法可受啟發，但所有素材與命名應為原創或具合法授權。

## 3. 使用者與使用情境

### 3.1 主要使用者

- 想快速遊玩 2～10 分鐘休閒遊戲的使用者。
- 使用滑鼠、鍵盤、觸控螢幕或手機的玩家。
- 中文、英文或日文使用者。
- 需要較大文字、清楚色彩或減少動態效果的使用者。

### 3.2 典型流程

1. 玩家雙擊 `index.html`。
2. 遊戲顯示語言選擇結果與主畫面；首次載入依瀏覽器語言自動選擇，預設後備語言為繁體中文。
3. 玩家按「開始遊戲」，畫面提示點擊／按鍵以解鎖音訊。
4. 遊戲開始，角色自動跳躍，玩家只控制左右方向。
5. 角色持續向上，鏡頭平滑捲動，玩家取得分數與道具。
6. 遊戲結束後輸入暱稱，分數寫入本機排行榜。
7. 玩家可重新挑戰、查看排行榜或返回主畫面。

### 3.3 回訪流程

- 若存在有效存檔，主畫面的「繼續遊戲」可用，並顯示「上次高度／時間」小型摘要。
- 按下「繼續遊戲」後還原角色、平台、道具、分數、隨機種子與難度狀態。
- 若存檔版本不相容或內容損壞，顯示可理解的提示，安全刪除該筆存檔並保留設定與排行榜。

## 4. 技術與執行限制

### 4.1 固定技術棧

- HTML5：頁面結構、對話框與 Canvas 容器。
- CSS3：RWD、主題、元件、動畫與安全區。
- 原生 JavaScript（ES2018 相容語法）：遊戲迴圈、資料管理、輸入、音訊、國際化與測試。
- Canvas 2D：遊戲場景、角色、平台、粒子、背景與 HUD。
- Web Audio API：音樂增益、音效混音與 limiter。
- `localStorage`：設定、暫停存檔與本機排行榜。

### 4.2 `file://` 相容策略

為確保直接雙擊即可執行，必須遵守下列規則：

- 不使用 `type="module"`、動態 `import()` 或需要 HTTP Origin 的模組載入方式。
- 不使用 `fetch()` 或 XHR 載入 JSON、語系、關卡、圖片資料。
- 語系與設定資料以一般 JavaScript 檔案註冊至單一命名空間。
- 所有腳本以 `<script defer src="...">` 依賴順序載入。
- 所有 CSS 以 `<link rel="stylesheet" href="...">` 載入。
- 圖片、音訊、字型使用相對路徑；不使用 `/assets/...` 這類站台根路徑。
- 不使用 CDN、遠端 API、遠端 Web Font、Service Worker 或後端。
- 若個別瀏覽器禁止 `file://` 下的持久儲存，遊戲仍可遊玩，並以 UI 提示該次工作階段無法保存。

### 4.3 JavaScript 組織方式

所有檔案使用 IIFE（立即執行函式）與唯一全域命名空間 `DJGame`：

```js
window.DJGame = window.DJGame || {};

(function (Game) {
  'use strict';
  Game.ScoreService = Object.freeze({
    calculate: function (state) { /* ... */ }
  });
}(window.DJGame));
```

限制：除 `window.DJGame` 與測試頁專用的 `window.DJTest` 外，不新增其他全域變數。

### 4.4 瀏覽器支援基準

- Chrome／Edge：最近 2 個主要版本。
- Firefox：最近 2 個主要版本。
- Safari／iOS Safari：最近 2 個主要版本。
- Android Chrome：最近 2 個主要版本。
- 最低畫面：320 × 568 CSS px。
- 最大常用畫面驗證：2560 × 1440 CSS px。
- 觸控：單點與同時按住左右控制鍵皆需正確處理；不依賴 hover 才能操作。

## 5. 導航與狀態流程

### 5.1 應用狀態

```text
BOOT
  ├─> HOME
  │    ├─> NEW_GAME -> PLAYING <-> PAUSED -> GAME_OVER
  │    ├─> CONTINUE -> PLAYING
  │    ├─> HELP -> HOME
  │    ├─> SETTINGS -> HOME
  │    └─> LEADERBOARD -> HOME
  └─> RECOVERABLE_ERROR -> HOME
```

### 5.2 狀態規則

- 同一時間僅能有一個頂層頁面可見。
- 切換頁面時要更新焦點，並讓螢幕閱讀器得知頁面標題。
- `PLAYING` 中按 `Escape`、`P`、頁面失焦或瀏覽器進入背景時切換成 `PAUSED`。
- 暫停時停止物理更新、分數時間與危險物生成，但保留畫面並降低 BGM 音量。
- `visibilitychange` 為 hidden 時立即寫入恢復存檔；禁止角色在背景分頁繼續掉落。
- 從遊戲返回主畫面前需建立暫存；開始新遊戲且已有存檔時顯示確認對話框。

## 6. 頁面與功能規格

### 6.1 主畫面 `HOME`

主畫面保持簡潔，只呈現品牌視覺、角色小動畫、版本號、存檔摘要與導航按鈕。不得直接放置音量滑桿、主題選單、語言下拉或鍵位設定。

必備按鈕：

1. **開始遊戲**：開始新的遊戲流程。
2. **繼續遊戲**：讀取上次有效存檔；無存檔時顯示停用樣式與「尚無存檔」說明。
3. **說明**：進入完整說明頁。
4. **設定**：進入設定頁。
5. **排行榜**：進入本機排行榜頁，使用獎盃圖示。

版面規則：

- 導航按鈕使用單欄或寬畫面 2 欄排列，每個觸控目標至少 52 × 52 px。
- Logo 不超過首屏高度的 25%，避免把按鈕推到畫面外。
- 背景可有緩慢雲朵、氣球、小星星與紙飛機；裝飾元素不可攔截點擊。
- 小動畫遵守「減少動態效果」設定。
- 主畫面不顯示任何可直接更改設定值的控制項。

### 6.2 遊戲畫面 `PLAYING`

畫面區塊由上到下／由內到外分為：

- **狀態列 HUD**：分數、高度、連擊、目前增益與暫停鍵。
- **遊戲 Canvas**：角色、平台、障礙、道具、背景與特效。
- **行動控制 Dock**：只在觸控／窄畫面顯示，放於 Canvas 有效區域之外。
- **桌機輔助側欄**：寬螢幕可顯示操作提示與增益資訊，不覆蓋 Canvas。

HUD 規則：

- 分數數字至少 24 px、標籤至少 18 px。
- HUD 可置於 Canvas 上方獨立列；若因極小高度需疊加，必須使用半透明高對比底板，且不得遮住角色附近的中央安全遊戲區。
- 增益效果以圖示＋剩餘秒數呈現，不單靠顏色。
- 暫停鍵至少 48 × 48 px，距離裝置邊緣含安全區至少 8 px。

### 6.3 暫停頁 `PAUSED`

- 顯示「繼續」、「重新開始」、「設定」、「返回主畫面」。
- 背景遊戲畫面加 45% 暗色遮罩與輕微模糊，但停止所有世界動畫。
- 進入設定後返回暫停頁，不得自動解除暫停。
- 「重新開始」與「返回主畫面」需二次確認，避免誤觸。

### 6.4 說明頁 `HELP`

使用乾淨的分區卡片與大量圖示，不使用密集長段落。至少包含：

- **快速開始**：3 步驟圖解（移動、踩平台、自動向上跳）。
- **控制方式**：鍵盤、觸控按鈕、可選裝置傾斜，各自有圖示與動畫示意。
- **計分方式**：高度分、道具分、連續踩踏與風險獎勵。
- **平台圖鑑**：一般、移動、破裂、彈簧、消失、雲朵平台。
- **道具圖鑑**：彈簧鞋、火箭背包、護盾、磁鐵、慢動作、幸運星。
- **危險圖鑑**：怪物、黑洞、尖刺平台、暴風雲。
- **存檔與排行榜**：何時自動保存、排行榜為本機資料的明確說明。
- **無障礙提示**：減少動態、色弱友善、高對比與關閉傾斜操作方式。

每個圖鑑項目包含圖示、名稱、1～2 句效果、持續時間／分數與避免方式。說明頁支援目錄錨點、返回頂端按鈕與鍵盤操作。所有圖示需有可翻譯文字標籤，裝飾圖則使用空白替代文字。

### 6.5 設定頁 `SETTINGS`

設定頁採用單欄分組卡片，桌面寬螢幕最多 2 欄。每組標題清楚，留白充足。設定不出現在主畫面。

#### 音訊

- 主音量：0～100，預設 70。
- BGM 音量：0～100，預設 55。
- 音效音量：0～100，預設 75。
- 靜音切換。
- BGM 曲目：自動輪播／曲目 1／曲目 2／曲目 3。
- 音量滑桿旁顯示目前數值與相符圖示，支援方向鍵微調。
- 提供「試聽音效」與「試聽 BGM」按鈕；切換頁面後停止試聽。

#### 操作

- 左移鍵：預設 `ArrowLeft`／`A`。
- 右移鍵：預設 `ArrowRight`／`D`。
- 暫停鍵：預設 `Escape`／`P`。
- 觸控按鈕左右位置交換。
- 裝置傾斜控制：預設關閉，啟用前說明並依法取得權限。
- 傾斜靈敏度：1～5，預設 3。
- 鍵位衝突時禁止儲存並以文字指出衝突來源。

#### 顯示與輔助

- 語言：繁體中文／English／日本語。
- 配色主題：粉彩晴空／糖果夕陽／薄荷森林／星夜霓虹。
- 高對比模式：預設關閉。
- 減少動態效果：預設跟隨作業系統，可手動覆寫。
- 粒子量：低／中／高，預設中；低階裝置可自動降級。
- 畫面震動：開／關，預設開；減少動態時預設關。

#### 資料

- 清除繼續遊戲存檔。
- 清除本機排行榜。
- 還原全部設定。
- 所有清除行為需顯示二次確認，清楚列出影響範圍。

### 6.6 排行榜 `LEADERBOARD`

- 顯示本機前 20 名。
- 欄位：名次、暱稱、分數、最高高度、日期、使用主題圖示。
- 前三名使用金／銀／銅獎牌圖示，但仍顯示數字，避免只靠顏色。
- 預設依分數降冪；同分依高度降冪，再依達成時間較早者優先。
- 最新一筆成績以外框與「NEW」文字標記。
- 空狀態顯示可愛獎盃插圖與「完成第一場遊戲後，成績會出現在這裡」。
- 提供「返回」與「清除排行榜」；清除需二次確認。
- 排行榜明確標示「僅儲存在此瀏覽器」。

### 6.7 遊戲結束 `GAME_OVER`

- 顯示本局分數、最高高度、最佳連擊、收集數與是否刷新紀錄。
- 若可進排行榜，顯示暱稱輸入；1～12 個可見字元，去除首尾空白與控制字元。
- 暱稱預設沿用上次輸入；空白時使用當前語言的「玩家」。
- 按鈕：再玩一次、查看排行榜、返回主畫面。
- 成績只可提交一次，重複點擊不得產生重複紀錄。

## 7. RWD 與輸入設計

### 7.1 遊戲畫布座標

- 邏輯座標固定為 420 × 720；所有物理運算使用邏輯座標，避免裝置尺寸改變遊戲難度。
- Canvas CSS 尺寸依容器等比例縮放，內部像素尺寸乘以 `min(devicePixelRatio, 2)`。
- 不拉伸畫面；額外空間放置背景裝飾、側欄或控制區。
- 監聽 `resize` 與 `orientationchange`，使用 100 ms debounce 更新版面。
- 尺寸改變只影響呈現，不重置遊戲、速度、平台或分數。

### 7.2 建議斷點與版面

| 模式 | 條件 | 版面 |
|---|---|---|
| 小型直向 | 寬度 `< 480px` | HUD、Canvas、底部控制 Dock 單欄排列 |
| 大型直向／平板 | `480px–899px` | Canvas 置中，控制 Dock 位於下方，可顯示較大按鍵 |
| 桌機 | 寬度 `>= 900px` | Canvas 置中，左右為資訊／裝飾側欄，鍵盤為主 |
| 低高度橫向 | 高度 `< 520px` 且橫向 | Canvas 置中，左右控制鍵分居 Canvas 外側安全欄 |

斷點應以可用空間與內容是否溢出為準，不依賴特定品牌裝置。

### 7.3 行動控制不遮擋規格

- **直向**：左右控制鍵位於 Canvas 下方的獨立 `.touch-controls` 容器，容器不使用覆蓋 Canvas 的 `position: absolute`。
- **橫向**：左鍵位於 Canvas 左側欄、右鍵位於右側欄，側欄與 Canvas 同列但不重疊。
- 若裝置過窄無法保留兩側欄，縮小 Canvas 至可容納控制鍵，不得把按鍵疊上遊戲內容。
- 每個按鍵建議 72～96 px，絕不低於 56 × 56 px；按住時可連續輸入。
- 使用 `touch-action: none` 防止操作時頁面捲動；只套用在控制鍵與 Canvas，不鎖死說明／設定頁捲動。
- 支援 `pointerdown`、`pointerup`、`pointercancel`、`lostpointercapture`；任何中斷都必須釋放方向狀態，避免角色持續移動。
- 控制列需考慮 `env(safe-area-inset-bottom)`，在瀏海、圓角與手勢橫條上保留安全距離。
- 暫停鍵不與左右鍵相鄰，降低誤觸。

### 7.4 鍵盤、觸控與傾斜

- 鍵盤：左右方向鍵或 A／D；按住連續移動；避免按鍵重複事件造成加速度倍增。
- 觸控：左右大型按鍵；同時按下視為互相抵銷。
- 傾斜：選用功能，需使用者主動開啟；具有中央死區與低通濾波，避免抖動。
- 最近一次有效輸入裝置決定操作提示；切換裝置不需重新開始。
- 在表單欄位輸入時停用遊戲快捷鍵。

### 7.5 防止頁面遮擋與溢出

- 使用 `100dvh`，並以 `100vh` 作後備。
- 遊戲頁禁止整頁非預期捲動；主畫面、說明、設定、排行榜允許正常垂直捲動。
- 所有對話框最大高度為 `calc(100dvh - 32px)`，內容區可捲動，固定操作列不遮住內容。
- 200% 文字縮放下仍可操作，不出現水平捲動或按鈕文字裁切。

## 8. 視覺、配色與字體

### 8.1 視覺方向

主題為「可愛天空旅行」。角色、平台、道具與圖示偏圓潤可愛；背景採較寫實的天空漸層、光暈、雲層景深、遠山剪影、晨昏色溫與細微氣候變化。前景互動物件保持較高飽和度與清楚輪廓，避免與擬真背景混在一起。

### 8.2 字體規格

- 使用專案內合法授權的圓體 WOFF2；找不到字型時依序回退至 `system-ui`, `Arial`, `sans-serif`。
- 繁體中文需包含完整常用字；英文與日文不可因字型缺字顯示方框。
- 正文：桌機 18～20 px，手機不得小於 18 px。
- 次要文字：不得小於 16 px。
- 按鈕：20～24 px，字重 700。
- 頁面標題：32～48 px；卡片標題：22～28 px。
- HUD 數字：24～32 px，使用等寬數字特性 `font-variant-numeric: tabular-nums`。
- 行高：正文至少 1.5；日文與中文長文至少 1.65。
- 不以全大寫呈現長句，不使用過細字重，不在高彩度背景直接放無底板文字。

### 8.3 圖示規格

- 使用原創 SVG 為主，角色／複雜場景可使用 WebP 或 PNG。
- 風格：圓角、粗輪廓、簡單表情、柔和高光與小型裝飾（星星、愛心、雲、糖果、花朵）。
- 互動圖示必須搭配文字或 `aria-label`；不得只用不明確符號。
- 同一功能在所有頁面使用同一圖示。
- 圖示至少提供正常、hover、active、disabled、focus-visible 狀態。
- 不使用 emoji 作為唯一正式圖示，避免跨平台樣式不一致；可將 emoji 當非必要裝飾。

### 8.4 擬真與豐富度

- 背景至少 4 層視差：天空色、遠景、雲層、近景微粒。
- 高度區間改變環境：草原清晨、晴空、黃昏雲海、星夜高空。
- 加入低成本現實感：柔和陰影、角色落地壓縮、平台材質、光線方向、雲朵透明度與高度霧。
- 天候效果：微風葉片、輕雨、雪花或流星；一次只啟用一種，且不得遮擋平台邊緣。
- 遊戲物件加 2～3 px 深色外框或高對比陰影，確保各主題中皆清楚可見。
- 背景裝飾不可形成看似可踩的平台，危險物不可使用與獎勵物相同的主要輪廓。

### 8.5 配色主題

| 主題 | 背景主色 | 表面色 | 主要操作色 | 警告色 | 氛圍 |
|---|---|---|---|---|---|
| 粉彩晴空 | `#BDEBFF` | `#FFF9F0` | `#FF5FA2` | `#C43C4E` | 清晨、雲朵、彩虹 |
| 糖果夕陽 | `#FFB38A` | `#FFF2DC` | `#7C4DFF` | `#9F2439` | 黃昏、糖果、氣球 |
| 薄荷森林 | `#BDEFD4` | `#F5FFF9` | `#007D74` | `#B93845` | 樹葉、花朵、小鳥 |
| 星夜霓虹 | `#171A3D` | `#292D5C` | `#65E6FF` | `#FF7A90` | 月亮、星星、流星 |

實際實作以 CSS Custom Properties 管理。每個主題都需驗證：

- 一般正文與背景對比至少 4.5:1。
- 大型文字至少 3:1。
- 焦點框、按鈕邊界、滑桿軌道等非文字元件至少 3:1。
- 成功、危險、停用與選取狀態同時具圖示／文字／形狀差異，不單靠顏色。
- 高對比模式覆寫所有主題，優先確保辨識度。

### 8.6 動畫原則

- UI 轉場 150～250 ms；不使用超過 500 ms 的阻塞動畫。
- 角色動作使用插值與 squash-and-stretch，但碰撞盒不隨視覺壓縮改變。
- 平台、道具與危險物的動畫頻率互相區分。
- `prefers-reduced-motion: reduce` 或遊戲設定啟用時：停用視差、震動、粒子爆發與大型位移轉場，保留必要狀態提示。

## 9. 核心遊戲玩法

### 9.1 核心循環

1. 角色自動反覆跳躍。
2. 玩家控制水平移動與空中修正。
3. 角色下降時踩到平台便再次彈跳。
4. 當角色高於鏡頭上方觸發線，世界向下捲動。
5. 程序式產生新平台、道具與危險。
6. 角色跌出畫面、撞上致命障礙或被怪物擊中且無護盾時結束。

### 9.2 建議物理常數

所有數值以 420 × 720 邏輯座標與秒為單位，最終需透過遊戲測試微調。

| 項目 | 建議值 |
|---|---:|
| 固定更新步長 | `1/120 s` |
| 最大累積補幀 | `0.25 s` |
| 重力 | `1850 px/s²` |
| 一般起跳速度 | `-780 px/s` |
| 水平加速度 | `1800 px/s²` |
| 最大水平速度 | `270 px/s` |
| 空中阻力 | `0.86`（依固定步長換算） |
| 左右穿牆 | 開啟；保留角色寬度緩衝 |
| 鏡頭觸發線 | 畫面高度 38% |
| 死亡線 | 畫面底部外 64 px |

物理必須使用固定 timestep，渲染使用插值；不得直接用不受限的畫面 `deltaTime` 更新物理。

### 9.3 碰撞規則

- 只在角色垂直速度向下時檢查踩踏平台。
- 使用上一幀腳底位置與本幀腳底位置做 swept crossing，避免高速穿透。
- 平台有效接觸需有至少 20% 角色寬度的水平重疊。
- 落地後把角色腳底校正到平台頂部，立即套用起跳速度。
- 單向平台只從上方碰撞；角色上升時可穿過。
- 一次更新只接受一個最高優先碰撞，防止重複彈跳與重複計分。
- 左右環繞時不得同時與兩側同一平台重複碰撞。

### 9.4 平台類型

| 類型 | 外觀／提示 | 行為 | 建議出現高度 |
|---|---|---|---:|
| 一般 | 綠色草地／軟墊 | 固定不動 | 0 m 起 |
| 移動 | 藍色＋雙箭頭 | 水平或小幅垂直往返 | 250 m 起 |
| 破裂 | 棕色裂紋 | 踩一次延遲 180 ms 後碎裂 | 450 m 起 |
| 彈簧 | 黃色線圈 | 跳躍速度乘 1.45 | 300 m 起 |
| 消失 | 紫色閃爍 | 踩後淡出 2 秒，再淡入 | 750 m 起 |
| 雲朵 | 白色透明邊緣 | 踩一次即散開 | 1,000 m 起 |

程序生成器必須保證：

- 從當前可落腳平台至少存在一條符合最大跳躍水平／垂直距離的可達路徑。
- 不把必經平台完全藏在 HUD 或畫面外。
- 初始 200 m 不生成致命障礙。
- 連續破裂／雲朵平台不得構成唯一可行路徑。
- 以可重現的 seed PRNG 生成，以便存檔、測試與除錯。

### 9.5 道具

| 道具 | 效果 | 時間／次數 | 視覺回饋 |
|---|---|---|---|
| 彈簧鞋 | 每次起跳速度 ×1.25 | 3 次 | 腳部彈簧、計次圖示 |
| 火箭背包 | 自動向上飛行且短暫無敵 | 2.8 秒 | 火焰尾跡、鏡頭穩定 |
| 護盾 | 抵擋一次怪物／危險碰撞 | 1 次／8 秒上限 | 透明泡泡、破裂音效 |
| 磁鐵 | 吸引一定範圍的星星 | 6 秒 | 弧形吸附線、倒數 |
| 慢動作 | 危險物與平台速度 ×0.65 | 4 秒 | 冷色邊框，不降低輸入反應 |
| 幸運星 | 立即加分並提高短期稀有道具率 | 立即／5 秒 | 星光爆發、分數浮字 |

### 9.6 危險與敵人

- **彈跳怪物**：在平台上左右巡邏，可從上方踩掉並加分，側面碰撞受傷。
- **飛行怪物**：依固定路徑飛行，出現前有方向警示。
- **黑洞**：具吸引範圍，中心致命；外圈需清楚顯示。
- **尖刺平台**：不能踩踏，輪廓與一般平台明顯不同。
- **暴風雲**：週期性水平吹動角色，吹動前有 0.8 秒提示。

生成規則需保留反應時間，不在角色當下位置或鏡頭剛出現處立刻造成無法避免的傷害。

### 9.7 難度曲線

- 0～300 m：教學區，平台寬、間距小、無致命敵人。
- 300～1,000 m：加入移動／破裂平台與少量怪物。
- 1,000～2,500 m：平台逐漸縮窄、移動加快、開始組合危險。
- 2,500 m 以上：難度緩慢增加但有上限，避免成為純隨機不可能局面。
- 每 500 m 給予短暫里程碑演出與保底安全平台。
- 生成器的可達性永遠優先於難度。

### 9.8 計分

```text
總分 = 最高高度分 + 道具分 + 敵人分 + 連擊分 + 里程碑分
```

- 高度分：每上升 1 邏輯公尺加 1 分，只計最高到達高度，不因上下反覆累加。
- 一般星星：50 分；幸運星：250 分。
- 從上方踩掉怪物：200 分。
- 連續踩踏不同平台且未向下掉超過一個畫面時累積連擊，最高倍率 ×3。
- 每 500 m 里程碑：500 分。
- 暫停、改變視窗尺寸與切換設定不影響分數。

## 10. 音訊規格

### 10.1 BGM

- 至少 3 首原創或合法授權的輕快鋼琴曲。
- 每首建議 60～150 秒，能無明顯斷點循環。
- 氛圍：明亮、輕盈、節奏清楚，不使用過重低頻或刺激高頻。
- 建議曲目：〈Morning Cloud Steps〉、〈Candy Balloon Waltz〉、〈Starlight Piano Hop〉。
- 曲目自動輪播時避免連續重複；記錄目前曲目與播放進度，暫停後可延續。
- 首次載入不自動發聲；必須在玩家點擊開始、繼續或試聽後建立／恢復 `AudioContext`，遵守瀏覽器自動播放限制。

### 10.2 音效

音效以短促、高音、輕脆、低延遲為原則，至少包含：

- 一般落地跳躍：木琴／玻璃珠清脆音。
- 彈簧：向上滑音。
- 收集星星：短促鈴鐺音。
- 道具啟動：明亮和弦。
- 護盾破裂：柔和玻璃碎裂音。
- 怪物受擊：可愛短音，不使用寫實暴力聲。
- 按鈕確認／返回：兩種可辨識 UI 音。
- 遊戲結束／刷新紀錄：下降音型／小型勝利音型。

同一音效短時間重複時可使用 3% 隨機音高變化；必須限制同時播放數，避免刺耳疊音。

### 10.3 BGM 放大 10 倍的安全實作

為精確符合「遊戲中的 BGM 音量放大為原來 10 倍」，音訊節點固定為：

```text
BGM Source -> BGM Gain（固定 boost = 10.0）
           -> BGM User Gain（設定滑桿）
           -> DynamicsCompressor/Limiter
           -> Master Gain
           -> Audio Destination

SFX Source -> SFX User Gain -> Master Gain -> Audio Destination
```

- BGM boost 節點值必須為 `10.0`，不可僅把 UI 百分比文字寫成 10 倍。
- 音訊素材在製作階段預留足夠 headroom；建議峰值不高於 -20 dBFS。
- boost 後必須經 limiter／compressor，輸出峰值不超過 -1 dBFS，避免削波、爆音與傷害聽力。
- 使用者音量採等感知曲線，例如 `userGain = (slider / 100)²`。
- 最終 BGM 增益概念式：`10 × userGain × masterGain`，再經 limiter。
- 「靜音」必須將 Master Gain 平滑 ramp 至 0，不銷毀原音量設定。
- 音量變化使用 30～80 ms ramp，避免 click／pop。
- 單元測試需驗證 10 倍節點、靜音、滑桿換算、暫停 ducking 與設定保存；人工測試需確認沒有可聞削波。

### 10.4 音訊失敗後備

- 音訊解碼失敗時顯示非阻塞提示，遊戲仍可進行。
- `AudioContext` 不可用時隱藏不適用的試聽功能並標示「此瀏覽器不支援音訊」。
- 切到背景分頁時暫停 BGM；返回後需由安全的使用者互動恢復時才恢復。

## 11. 多國語系

### 11.1 支援語言

- `zh-TW`：繁體中文，預設後備語言。
- `en-US`：英文。
- `ja-JP`：日文。

### 11.2 實作方式

- 語系檔位於 `js/i18n/locales/`，以一般 JavaScript 檔案呼叫 `DJGame.I18n.register(locale, dictionary)`。
- 禁止以 `fetch()` 載入 JSON，以維持 `file://` 相容。
- HTML 使用 `data-i18n="home.start"`；含變數字串使用具名插值，如 `{score}`。
- 不以字串串接組句；各語言保留自己的語序。
- 數字與日期使用 `Intl.NumberFormat`、`Intl.DateTimeFormat`；若瀏覽器不支援則使用固定後備格式。
- 語言切換立即更新目前頁面、`document.documentElement.lang`、頁面標題、ARIA 標籤與 Canvas 內 HUD。
- Canvas 不直接保存翻譯結果；語言切換後重新取得字串。

### 11.3 語言選擇順序

1. 已保存的使用者語言。
2. `navigator.languages` 中第一個可支援語言。
3. 繁體中文 `zh-TW`。

### 11.4 翻譯完整性

- 三個語系 key 集合必須完全一致。
- 缺少 key 時開發模式顯示 `[missing:key]`，正式模式回退至繁體中文並記錄警告。
- 測試最長常用字串在 320 px 寬與 200% 文字縮放下不裁切。
- 英文按鈕不得為了塞入版面而縮小到 18 px 以下，應改用換行或加寬容器。

### 12.3 繼續遊戲存檔結構

- 名稱只作純文字顯示，使用 `textContent`，不得插入 `innerHTML`。
- 新成績加入後排序、去除第 21 名之後資料，再一次性寫回。
- 利用本局唯一 `submissionId` 防止同局重複提交。
- 排行榜損壞時先隔離該資料，顯示重置提示，不影響遊戲開始。
- 由於沒有後端，本機排行榜不宣稱防作弊或跨裝置同步。

### 12.5 儲存失敗

- 所有讀寫包在 `try/catch`，處理無痕模式、容量不足與儲存被封鎖。
- 寫入失敗時顯示一次非阻塞 toast，不可每幀重複提示。
- 遊戲改以記憶體狀態繼續，主流程不得崩潰。

## 13. 無障礙與易用性

- 所有可操作 DOM 元件皆可由鍵盤操作，焦點順序與視覺順序一致。
- 使用清楚的 `:focus-visible` 外框，任何主題不得移除焦點提示。
- 對話框開啟時焦點被限制在其中；關閉後回到原觸發按鈕。
- 圖示按鈕需有 `aria-label`，滑桿有可見標籤與目前數值。
- 頁面切換時更新具 `aria-live="polite"` 的頁面狀態；遊戲中每秒變動的分數不持續朗讀。
- 遊戲開始、暫停、結束、刷新紀錄可用簡短 live region 通知。
- 不以閃爍超過每秒 3 次的效果呈現任何內容。
- 支援高對比、減少動態、關閉畫面震動與降低粒子。
- 重要操作不要求精準拖曳；音量滑桿同時支援點擊與鍵盤。
- 說明頁提供所有玩法資訊，遊戲不假設玩家只靠顏色或聲音理解狀態。

## 14. 效能與穩定性

### 14.1 效能目標

- 目標更新／呈現：60 FPS；高更新率螢幕仍以固定物理步長維持一致速度。
- 中階手機遊玩 10 分鐘：P95 frame time 小於 20 ms，不應有持續低於 50 FPS 的區段。
- 首次可互動：本機一般 SSD／手機儲存環境小於 2 秒。
- Canvas 內同時存在物件建議上限：平台 40、敵人 12、道具 15、粒子 200（高）／80（中）／30（低）。
- 不在遊戲主迴圈中頻繁建立大型陣列、解析 JSON 或修改大量 DOM。
- 圖片採 sprite atlas 或預載快取；音訊預解碼後重用 buffer。

### 14.2 遊戲迴圈

- 使用 `requestAnimationFrame` 呈現。
- 使用 accumulator 固定步長更新，單幀最多執行限定次數；超過最大累積時間需丟棄多餘時間，避免死亡螺旋。
- 不以 `setInterval` 驅動物理。
- 分頁隱藏時停止 RAF 邏輯並暫停音訊。
- 對物件、粒子與音效 voice 使用 pool，避免高頻垃圾回收。
- 對離開鏡頭太遠的物件進行回收。

### 14.3 自動品質降級

若連續 3 秒平均 FPS 低於 45：

1. 將粒子量從高降為中或從中降為低。
2. 停用非必要遠景視差與模糊。
3. 降低天候粒子數。
4. 保留遊戲碰撞、輸入、HUD 與必要動畫。

品質恢復不可在短時間頻繁往返，至少觀察 15 秒後才升級，並尊重使用者手動選擇的低粒子模式。

### 14.4 穩定性

- 所有事件監聽器在離開頁面或銷毀模組時解除。
- 重開 20 局後，活動 RAF、音訊 source 與 pointer listener 數量不得累積。
- 音訊、圖像或字型其中一項載入失敗時仍可進入遊戲。
- 任何未處理例外不可讓主畫面永久空白；頂層錯誤邊界顯示可回首頁的錯誤卡片。

## 16. 模組介面與責任邊界

### 16.1 主要模組

| 模組 | 輸入 | 輸出／責任 | 禁止事項 |
|---|---|---|---|
| `StateMachine` | event、目前狀態 | 合法狀態轉換 | 不繪圖、不播音 |
| `GameLoop` | timestamp | 固定更新與插值比例 | 不含具體物理規則 |
| `Physics` | entity、dt、input | 新位置與速度 | 不讀 DOM |
| `Collision` | 玩家、平台／危險 | 碰撞結果 | 不直接加分 |
| `WorldGenerator` | seed、高度、難度 | 可達的物件集合 | 不使用 `Math.random()` |
| `Score` | 遊戲事件 | 分數與連擊 | 不操作 UI |
| `Renderer` | 唯讀遊戲快照、alpha | Canvas 畫面 | 不改遊戲快照 |
| `InputManager` | 鍵盤／pointer／tilt | 正規化 action state | 不直接移動角色 |
| `AudioManager` | 音訊事件、設定 | 混音與播放 | 不控制遊戲狀態 |
| `Storage` | key、validated data | 安全 JSON 讀寫 | 不信任原始資料 |
| `I18n` | locale、key、params | 翻譯文字 | 不 fetch |
| `Router` | route event | DOM 頁面切換與焦點 | 不保存遊戲物理 |

### 16.2 事件名稱

事件使用 `domain:action` 形式，至少包含：

- `app:ready`, `route:changed`
- `game:started`, `game:paused`, `game:resumed`, `game:over`
- `player:landed`, `player:damaged`, `player:died`
- `item:collected`, `enemy:defeated`, `milestone:reached`
- `score:changed`, `save:completed`, `save:failed`
- `settings:changed`, `locale:changed`, `audio:unlocked`

高頻逐幀資料不可透過全域事件匯流排傳遞，避免不必要配置與難以追蹤的時序。

## 17. 資產規格

### 17.1 圖像

- 角色至少具 idle、上升、下降、落地、受傷、道具與勝利表情。

- BGM 可提供 MP3 與 OGG 兩種格式以提高瀏覽器相容性；由程式選擇可播放格式。
- SFX 建議短音使用 WAV 或 OGG，並控制檔案大小。
- 音訊檔名使用小寫 kebab-case，不含空白與非 ASCII 字元。
- BGM 原始檔峰值需符合 §10.3 的 10 倍增益 headroom。

### 17.3 字型

- 只放實際使用字重，避免大型不必要字型。
- 設定 `font-display: swap`。
- 字型失敗時排版仍不溢出，按鈕尺寸不可依賴單一字型精確寬度。

## 19. 測試規格

### 19.1 測試執行方式

- 直接雙擊 `tests/test-runner.html` 即可執行，不需要 npm、build 或 server。
- 測試框架使用專案內自製的輕量 harness，不依賴 CDN。
- 頁面顯示總數、通過、失敗、耗時與每個失敗的預期／實際結果。
- 支援「全部執行」與依模組執行。
- 測試結果不得依賴執行順序；每個案例需 arrange／act／assert 並自行清理。
- 對時間、PRNG、儲存與 AudioContext 使用可注入 fake／stub，確保結果可重現。

### 19.2 單元測試清單

#### 物理與碰撞

- 固定 timestep 在 30／60／120 Hz 呈現下產生相近結果。
- 重力、最大水平速度、阻力與左右穿牆正確。
- 上升穿過平台不碰撞，下降跨越平台頂面可落地。
- 高速下降不穿透平台。
- 邊緣最小重疊規則正確。
- 同幀多平台只觸發一次落地與一次分數事件。
- 破裂、彈簧、消失、雲朵平台狀態轉換正確。

#### 世界生成與難度

- 同一 seed 產生完全相同序列。
- 不同 seed 有合理差異。
- 每批生成至少有一條可達路徑。
- 各高度解鎖正確的平台、敵人與道具。
- 教學區不生成致命障礙。
- 難度達上限後不持續失控增加。

#### 道具、敵人與計分

- 每項道具啟用、倒數、疊加／不疊加與結束邏輯正確。
- 護盾只抵擋一次有效攻擊。
- 慢動作不改變輸入採樣。
- 高度只對新最高值計分。
- 連擊開始、中斷、倍率上限正確。
- 同一收集物與怪物不重複加分。
- 里程碑僅觸發一次。

#### 狀態與存檔

- 所有合法狀態轉換可執行，非法轉換被拒絕。
- 分頁隱藏與失焦會暫停。
- 存檔 round-trip 後關鍵狀態一致。
- 損壞 JSON、錯誤 schema、NaN、Infinity、過大陣列被安全拒絕。
- 儲存被封鎖時回退記憶體模式且不崩潰。
- 開新局、死亡與清除存檔行為正確。

#### 排行榜

- 依分數、高度、時間規則排序。
- 僅保留前 20 名。
- 同一 `submissionId` 不重複加入。
- 暱稱長度、空白、控制字元與 HTML 字串處理正確。
- 清除只刪除排行榜 key。
- 損壞資料可復原且不影響其他儲存。

#### 輸入與 RWD 邏輯

- 鍵盤按下／放開與失焦重置正確。
- 左右同時輸入互相抵銷。
- `pointercancel` 與失去 capture 後方向歸零。
- 表單聚焦時快捷鍵不作用。
- 各斷點選出正確布局模式。
- 旋轉／縮放後邏輯座標與遊戲狀態不變。
- 安全區與控制 Dock 計算結果不侵入 Canvas 矩形。

#### 音訊

- 使用者第一次互動才解鎖音訊。
- BGM 固定 boost GainNode 等於 `10.0`。
- 使用者音量曲線、Master／BGM／SFX 聲道組合正確。
- 靜音與解除靜音保留先前數值。
- 暫停 ducking、切背景暫停與恢復正確。
- voice 上限與短時間重複音效限制正確。
- 音訊初始化失敗不阻止遊戲。

#### 語系、設定與 UI 狀態

- 三語 key 集合完全一致。
- 缺 key 正確回退繁體中文。
- 插值不遺失參數、不執行 HTML。
- 語言偵測與保存優先順序正確。
- 設定預設值、範圍校正與還原正確。
- 鍵位衝突被拒絕。
- 無存檔時繼續按鈕停用，有有效存檔時啟用。
- 同局成績提交按鈕連點仍只加入一筆。

| 輸入 | 鍵盤、滑鼠、單點觸控、多指觸控、旋轉、失焦 |
| 輔助 | 高對比、減少動態、粒子低、靜音、鍵盤-only |
| 持久化 | 正常、儲存被拒、損壞 JSON、舊 schema、容量不足 |

### 19.4 長時間與效能測試

- 連續遊玩 30 分鐘，無明顯記憶體持續成長。
- 連續開始／結束 20 局，事件監聽器、RAF、音訊 voice 不累積。
- 模擬 30 FPS 與 120 FPS 顯示，物理速度與跳躍高度保持一致。
- 快速切換背景／前景 20 次，遊戲不偷跑、不重複播放 BGM。
- 快速旋轉直向／橫向 10 次，控制鍵永遠不遮 Canvas。
- 粒子高設定下壓力測試，觸發品質降級後仍維持輸入反應。

### 19.5 測試通過門檻

- 核心純函式與資料模組單元測試全部通過。
- 不接受已知 Critical／High 嚴重度缺陷。
- 三語系無 missing key。
- RWD 矩陣不得出現遊戲控制遮住有效畫面的情況。
- 10 倍 BGM 節點測試通過，且實機聆聽無削波、爆音。
- 直接以 `file://` 開啟的遊戲與測試頁皆可運作。

## 23. 風險與對策

| 風險 | 影響 | 對策 |
|---|---|---|
| `file://` 對模組／資料載入限制 | 遊戲無法啟動 | 使用 classic defer scripts、JS 語系註冊、相對資產路徑，不使用 fetch |
| 瀏覽器禁止自動播放 | 初次沒有 BGM | 首次明確互動後才解鎖 AudioContext，顯示音訊提示 |
| BGM 10 倍造成削波或過大音量 | 聽感差、聽力風險 | 素材預留 -20 dBFS headroom、Gain=10 後接 limiter、預設使用者音量 55% |
| `localStorage` 在部分本機環境不可用 | 無法存檔／排行 | 功能偵測、try/catch、記憶體後備與非阻塞提示 |
| 手機瀏覽器動態工具列改變高度 | UI 被裁切 | 使用 `100dvh`＋後備、safe-area 與 resize 重新布局 |
| 橫向手機高度過小 | 控制與 HUD 擁擠 | 左右安全欄、Canvas 等比縮小、精簡但不縮小必要字體 |
| 可愛細節過多降低辨識度 | 平台／危險看不清 | 物件輪廓、背景降對比、粒子上限、高對比模式 |
| 程序生成出現不可能路徑 | 不公平死亡 | seed PRNG、可達性驗證、保底平台與生成器單元測試 |
| 長時間遊戲資源累積 | 卡頓或崩潰 | 物件池、回收離屏物件、解除 listener、20 局壓力測試 |
| 大型中日文字型拖慢載入 | 首次畫面延遲 | 字型子集化、只留必要字重、`font-display: swap` |
