## 1. 專案概述

### 1.1 目標

打造一款以標準 Tetris 規則為基礎的 Web 瀏覽器俄羅斯方塊遊戲，提供流暢的單人遊戲體驗、完整的音效系統、清晰的介面設計，並支援鍵盤與觸控操作。

### 1.2 核心功能清單

| 功能類別 | 功能項目 |
|----------|----------|
| 遊戲核心 | 七種標準 Tetromino、旋轉、移動、消行、Ghost Piece |
| 畫面 | 主選單、遊戲畫面、暫停畫面、結算畫面、設定畫面 |
| 計分 | 即時分數、消行加分、連消 Combo、排行榜（本地） |
| 難度 | 20 個關卡，速度隨關卡提升 |
| 音效 | 背景音樂（BGM）、操作音效、消行音效、遊戲結束音效 |
| 特效 | 消行閃光、方塊落下特效、Level Up 動畫 |
| 儲存 | 最高分、設定值（localStorage） |
| 設定 | 音量控制、BGM 開關、音效開關、Ghost Piece 開關 |
| 操作 | 鍵盤、行動裝置觸控虛擬按鍵 |

## 2. 技術架構

### 2.1 技術選型

| 項目 | 選擇 | 說明 |
|------|------|------|
| 語言 | HTML5 / CSS3 / JavaScript (ES6+) | 無框架依賴，原生實作 |
| 渲染方式 | HTML5 Canvas 2D API | 遊戲主畫面渲染 |
| UI 層 | DOM + CSS | 介面、選單、按鈕 |
| 音效 | Web Audio API | 音效產生與控制 |
| 儲存 | localStorage | 高分與設定持久化 |
| 打包 | 無（單一 HTML 或多檔案） | 可選用 Vite 作為開發工具 |

### 2.2 遊戲主迴圈架構

```
requestAnimationFrame Loop
  └─ update(deltaTime)
       ├─ 處理輸入佇列
       ├─ 計算方塊下落 (gravity tick)
       ├─ 碰撞偵測
       ├─ 消行邏輯
       ├─ 分數更新
       └─ 渲染 (render)
            ├─ 清除 Canvas
            ├─ 繪製遊戲板
            ├─ 繪製 Ghost Piece
            ├─ 繪製當前方塊
            ├─ 繪製側邊資訊
            └─ 繪製 UI 覆蓋層
```

### 2.3 狀態機

遊戲有以下主要狀態，每次只能存在一種：

```
MENU → PLAYING ⇄ PAUSED
                ↓
             GAME_OVER → MENU
```

| 狀態 | 說明 |
|------|------|
| MENU | 主選單畫面 |
| PLAYING | 遊戲進行中 |
| PAUSED | 遊戲暫停 |
| GAME_OVER | 遊戲結束結算 |

## 3. 遊戲規則與核心機制

### 3.1 遊戲板（Board）

- **尺寸：** 10 欄 × 20 列（標準 Tetris 規格）
- **隱形緩衝區：** 上方額外保留 2 列（不顯示），供方塊生成用
- **座標系：** 左上角為原點 (0,0)，X 向右，Y 向下

### 3.2 方塊生成

- 使用 **7-bag 隨機系統**：將七種方塊隨機洗牌後依序取出，確保每 7 顆必出現所有類型，避免長時間不出現特定方塊。
- 每次取出一顆方塊後，立即預覽下一顆（Next Piece）。
- 顯示接下來 **3 顆** 預覽方塊（NEXT QUEUE）。

### 3.3 方塊下落

- 方塊按照固定時間間隔（Gravity）自動向下移動一格。
- 下落速度隨關卡提升（詳見第 8 節）。
- 玩家可按「快速下落」鍵加速方塊下落（Soft Drop，速度為正常的 20 倍）。
- 玩家可按「硬落」鍵讓方塊立即落到底部並鎖定（Hard Drop）。

### 3.4 鎖定延遲（Lock Delay）

- 方塊觸底後，給予 **0.5 秒**的鎖定延遲。
- 在延遲期間，玩家仍可移動或旋轉方塊。
- 每次移動/旋轉後，鎖定延遲重置，但同一顆方塊最多重置 **15 次**，超過後強制鎖定。

### 3.5 旋轉系統（Super Rotation System，SRS）

- 採用業界標準的 **SRS（Super Rotation System）**旋轉系統。
- 支援 **Wall Kick**：當旋轉碰到邊界或其他方塊時，系統嘗試依序試移 4 個候補位置，若有任一成功則完成旋轉。
- 每個方塊有 4 種旋轉狀態（0°, 90°, 180°, 270°）。
- SRS Wall Kick 偏移表依照官方規格實作。

### 3.6 Hold 功能

- 玩家可按 Hold 鍵將當前方塊暫存，並立即換出已暫存的方塊（若有）。
- 若 Hold 區為空，則當前方塊放入 Hold，並從 Next Queue 取出下一顆。
- 每顆方塊落地前只能使用 **一次** Hold。

### 3.7 消行邏輯

- 當一整列填滿時觸發消行。
- 多行同時消除時，視為一次消行事件，根據消除行數計分。
- 消行後，上方所有方塊向下掉落填補空缺（Gravity Fill）。
- 若消行後造成懸空的方塊群，不進行額外的「重力連鎖」（即標準 Tetris 規則）。

### 3.8 T-Spin 偵測

- 偵測是否為 T-Spin（T 形方塊使用旋轉進入的消行方式）：
  - **T-Spin Single / Double / Triple** 的計分為普通消行的 3 倍。
- 偵測條件：使用三角判定法（3-corner rule），最後動作必須是旋轉。

### 3.9 遊戲結束條件

- 新方塊生成位置與已有方塊重疊（堆疊超過遊戲板頂端），即判定遊戲結束（Block Out）。

## 5. 方塊（Tetromino）規格

### 5.1 七種方塊定義

| 名稱 | 形狀 | 顏色 | Hex |
|------|------|------|-----|
| I | ████ | 青色（Cyan） | `#00F0F0` |
| O | ██<br>██ | 黃色（Yellow） | `#F0F000` |
| T | ███<br>_█ | 紫色（Purple） | `#A000F0` |
| S | _██<br>██ | 綠色（Green） | `#00F000` |
| Z | ██_<br>_██ | 紅色（Red） | `#F00000` |
| J | █__<br>███ | 藍色（Blue） | `#0000F0` |
| L | __█<br>███ | 橘色（Orange） | `#F0A000` |

### 5.2 方塊繪製規格

每個方塊格子以 32×32 px 渲染，繪製方式如下：

- **外框：** 方塊主色，2px 框線
- **亮面（左上）：** 主色調亮 30%（模擬 3D 立體感）
- **暗面（右下）：** 主色調暗 30%
- **填充：** 主色

### 5.3 Ghost Piece

- Ghost Piece 為當前方塊垂直落下的預覽位置。
- 以半透明（opacity: 0.25）且輪廓線方式呈現（填色改為透明，只顯示 border）。
- 可在設定中關閉。

### 5.4 方塊生成位置

- 方塊於遊戲板頂端中央（第 4~7 欄）生成，初始旋轉狀態為 0°。
- I 形方塊生成於隱形緩衝區的第 21 列（距頂 1 格）。
- 其他方塊生成於第 21~22 列。

## 6. 操作與控制

### 6.1 鍵盤操作

| 按鍵 | 功能 |
|------|------|
| `←` / `A` | 向左移動 |
| `→` / `D` | 向右移動 |
| `↓` / `S` | 軟落（Soft Drop，加速下落） |
| `↑` / `W` / `X` | 順時針旋轉 |
| `Z` / `Ctrl` | 逆時針旋轉 |
| `Space` | 硬落（Hard Drop） |
| `C` / `Shift` | Hold（暫存方塊） |
| `P` / `Esc` | 暫停 / 繼續 |
| `R` | 重新開始（需確認） |

**DAS（Delayed Auto Shift）設定：**

| 項目 | 預設值 |
|------|--------|
| DAS 延遲（首次按住觸發自動重複前的等待） | 167ms |
| ARR（Auto Repeat Rate，重複移動間隔） | 33ms |
| SDF（Soft Drop Factor）| 20× |

### 6.2 行動裝置虛擬按鍵

在觸控裝置上，遊戲板下方顯示虛擬方向鍵：

```
           [ ↑ 旋轉 ]
[ ←左 ]  [ ↓軟落 ]  [ →右 ]
[ HOLD ]  [ ██硬落██ ]  [ PAUSE ]
```

- 虛擬按鍵支援長按觸發 DAS/ARR。
- 觸控目標最小尺寸：48×48px（符合無障礙設計）。
- 按鈕半透明，不遮擋遊戲視野。

### 6.3 滑動手勢（選配）

| 手勢 | 功能 |
|------|------|
| 左滑 | 向左移動 |
| 右滑 | 向右移動 |
| 下滑（快） | 硬落 |
| 下滑（慢） | 軟落 |
| 上滑 | 旋轉 |
| 雙指點擊 | 暫停 |

## 7. 計分系統

### 7.1 消行分數

消行分數以當前關卡數作為乘數：

| 消除行數 | 名稱 | 基礎分數 | 計算公式 |
|----------|------|----------|----------|
| 1 行 | Single | 100 | 100 × Level |
| 2 行 | Double | 300 | 300 × Level |
| 3 行 | Triple | 500 | 500 × Level |
| 4 行 | Tetris | 800 | 800 × Level |

### 7.2 T-Spin 分數

| 消除行數 | 名稱 | 基礎分數 | 計算公式 |
|----------|------|----------|----------|
| 0 行 | T-Spin | 400 | 400 × Level |
| 1 行 | T-Spin Single | 800 | 800 × Level |
| 2 行 | T-Spin Double | 1200 | 1200 × Level |
| 3 行 | T-Spin Triple | 1600 | 1600 × Level |

### 7.3 Combo 連消加分

連續消行（每次落下方塊都有消行）觸發 Combo：

| Combo 次數 | 額外加分 |
|------------|----------|
| 1 Combo | 50 × Level |
| 2 Combo | 100 × Level |
| n Combo | 50 × n × Level |

### 7.4 Hard Drop 與 Soft Drop 獎勵

| 動作 | 每格獎勵 |
|------|----------|
| Soft Drop（每格） | 1 分 |
| Hard Drop（每格） | 2 分 |

### 7.5 Back-to-Back 獎勵

連續出現「Tetris」或「T-Spin」（之間不穿插普通消行），觸發 Back-to-Back，消行分數乘以 1.5 倍。

## 8. 關卡與難度系統

### 8.1 關卡提升條件

每消除 **10 行**，關卡提升一級（最高 Level 20）。

### 8.2 下落速度表（Gravity）

下落速度以「每格下落所需幀數（60fps）」或「每格下落秒數」表示：

| 關卡 | 下落間隔（秒） |
|------|----------------|
| 1 | 1.000 |
| 2 | 0.793 |
| 3 | 0.618 |
| 4 | 0.473 |
| 5 | 0.355 |
| 6 | 0.262 |
| 7 | 0.190 |
| 8 | 0.135 |
| 9 | 0.094 |
| 10 | 0.064 |
| 11 | 0.043 |
| 12 | 0.028 |
| 13 | 0.018 |
| 14 | 0.011 |
| 15 | 0.007 |
| 16~20 | 0.005（最速） |

速度公式參考：`interval = (0.8 - ((level - 1) * 0.007))^(level - 1)` 秒

### 8.3 Level Up 提示

- 關卡提升時，畫面中央短暫顯示「LEVEL UP!」動畫（約 1.5 秒）。
- 播放 Level Up 音效。
- 遊戲板邊框短暫閃光。

## 9. 音效系統

### 9.1 技術實作

使用 **Web Audio API** 搭配 **AudioContext** 實作，音效分為兩個音軌：

- **BGM 音軌（GainNode）：** 控制背景音樂音量，loop 播放。
- **SFX 音軌（GainNode）：** 控制音效音量，短暫播放後自動釋放。

若瀏覽器因自動播放政策（Autoplay Policy）禁止音訊，在玩家首次互動（點擊/按鍵）後再啟動 AudioContext。

### 9.2 音效檔案清單

| ID | 觸發時機 | 格式 | 時長（約） |
|----|----------|------|------------|
| `bgm_main` | 遊戲進行中循環播放 | OGG / MP3 | 60–120 秒 |
| `bgm_menu` | 主選單循環播放 | OGG / MP3 | 30–60 秒 |

### 9.3 BGM 行為

| 情境 | 行為 |
|------|------|
| 進入主選單 | 播放 `bgm_menu`，loop |
| 開始遊戲 | 切換至 `bgm_main`，loop（淡入 0.5 秒） |
| 暫停 | BGM 音量降至 30%（淡出 0.3 秒） |
| 繼續遊戲 | BGM 音量恢復至設定值（淡入 0.3 秒） |
| 遊戲結束 | BGM 停止，播放 `sfx_gameover` |
| 關卡 15 以上 | 播放速度加快至 1.1 倍（選配） |

### 9.4 音效優先級與同時播放

- SFX 支援多聲道同時播放（透過每次建立新的 AudioBufferSourceNode）。
- `sfx_move` 若播放頻率過高（DAS 期間），設定去抖動：最短間隔 30ms 才播放一次。
- `sfx_tetris`、`sfx_tspin` 播放時，`sfx_clear_x` 不重疊播放。

## 10. 視覺特效

### 10.1 消行特效

1. **閃光（Flash）：** 被消除的行在消除前整排變白並閃爍（2 次，持續 0.15 秒）。
2. **消行動畫：** 消除行從中心向兩側收縮並消失（持續 0.1 秒）。
3. **粒子效果（選配）：** 消除行爆炸出小方塊粒子，受重力下落後消失。

### 10.2 方塊鎖定特效

- 方塊鎖定時，閃爍亮白一次（持續 0.05 秒）。

### 10.3 Hard Drop 特效

- Hard Drop 時，方塊沿下落路徑留下「殘影」（Trail），快速淡出。

### 10.4 Combo / Tetris / T-Spin 提示

- 畫面左側（或中央）浮現文字提示，如：
  - `TETRIS!`
  - `T-SPIN DOUBLE!`
  - `2 COMBO!`
  - `BACK-TO-BACK!`
- 文字從下方飛入，停留 1.5 秒後淡出。
- Tetris 提示使用金色，T-Spin 使用紫色，Combo 使用橘色。

### 10.5 Level Up 動畫

- 畫面中央顯示「LEVEL UP」+ 新關卡號碼，搭配放大縮小動畫（Scale Bounce）。
- 遊戲板邊框快速閃爍 3 次。

### 10.6 Game Over 動畫

- 方塊從最下方一列開始，逐列將格子變為灰色（由下而上，每列間隔 0.05 秒）。
- 全灰後顯示結算畫面。

## 14. 錯誤處理與邊界條件

| 情境 | 處理方式 |
|------|----------|
| 瀏覽器不支援 Canvas | 顯示提示文字：「您的瀏覽器不支援 Canvas，請更新瀏覽器。」 |
| 瀏覽器不支援 Web Audio API | 靜音繼續，不顯示錯誤 |
| localStorage 不可用 | 遊戲正常運作，不儲存記錄，結算畫面說明無法儲存 |
| 分頁切換至背景 | 自動暫停（監聽 `document.visibilitychange`） |
| 視窗縮放 | 遊戲板依比例重新縮放，不中斷遊戲 |
| 快速按鍵重複輸入 | 依 DAS/ARR 設定控制，防止過度觸發 |
| 方塊生成位置被遮住 | 判定 Block Out，結束遊戲 |

## 15. 效能需求

| 指標 | 目標值 |
|------|--------|
| 幀率（Frame Rate） | 穩定 60 FPS |
| 輸入延遲（Input Latency） | < 16ms（一幀內回應） |
| 首次載入時間（FCP） | < 2 秒（3G 網路） |
| 記憶體使用 | < 50MB |
| 音效延遲（Audio Latency） | < 20ms（Web Audio API 緩衝） |

**優化策略：**

- 方塊繪製使用 Canvas 而非 DOM，避免頻繁 Reflow。
- 音效使用 `AudioBuffer` 預載，避免播放時延遲。
- 粒子特效使用物件池（Object Pool）管理，避免頻繁 GC。
- 使用 `requestAnimationFrame` 而非 `setInterval` 驅動遊戲迴圈。
